# Echo Desktop Style Guide

本文件用于约束 Echo 桌面端的视觉风格、布局规则、交互复杂度和任务化设计原则。

Echo 的目标不是做成一个复杂工作台，而是一个低负担、高明确性、具有持续推进感的桌面产品。

## 1. 设计目标

Echo 需要同时满足以下目标：

- 简单好用
- 功能明确
- 减少冗余
- 可持续推进
- 具备轻游戏化的任务驱动感
- 能直接映射真实前后端能力

Echo 不是：

- 聊天网页
- 重型任务管理器
- 分析大屏
- 花哨的游戏界面

Echo 应该像一个安静、克制、可靠的桌面陪伴式执行界面。

## 2. 总体原则

### 2.1 简单优先

- 一个页面只服务一个核心目标
- 一个区域只表达一种主要信息
- 一个时刻只鼓励一个主动作

### 2.2 默认收敛

- 默认只显示必要内容
- 历史、解释、详情进入次层或详情层
- 不在主页面堆叠大量辅助信息

### 2.3 图形优先于长文本

- 能用波形、进度条、分段条、趋势线、网格表达的，不写成长段说明
- 文本负责命名、澄清、反馈，不负责重复视觉信号

### 2.4 苹果式秩序

- 布局平衡
- 对齐严格
- 留白克制
- 原生控件感强
- 层级清晰

### 2.5 任务驱动感

Echo 可以借用游戏任务系统的设计理念，但视觉仍保持克制、桌面化。

借用的不是“游戏皮肤”，而是：

- 明确目标
- 明确步骤
- 明确完成条件
- 明确进度反馈
- 明确推进感

### 2.6 长期陪伴优先

Echo 的核心不是“派发任务”，而是通过对话、记忆、反思，逐渐成为另一个“我”。

因此界面需要持续传达以下感受：

- 被理解
- 被记住
- 被陪伴
- 被温和地推动

任何任务化、进度化、成就化设计，都必须服从这个目标，不能反过来把 Echo 做成冰冷的效率工具。

## 3. 复杂度边界

### 3.1 交互层级

全局只保留三层：

1. 主层：当前页面最重要内容和主动作
2. 次层：补充模块、辅助列表、轻趋势信息
3. 详情层：侧板、弹层、展开内容

禁止：

- 四层及以上深层跳转
- 弹层里再开弹层
- 一个页面承载多个互相竞争的主流程

### 3.2 页面数量

桌面端首版只围绕五个核心页面：

1. 此刻
2. 学习
3. 行动
4. 反思
5. 记忆

后续如需扩展成就页、个人档案页、成长页，应视作二期内容。

### 3.3 模块数量

- 每个主页面最多 3 到 5 个核心模块
- 同屏不要出现多个同等强度的视觉重点
- 每个局部区域最多一个主按钮
- 右侧状态卡不得无限扩展

## 4. 布局规则

### 4.1 全局框架

全局始终保留左侧导航轨。

基础框架：

- 左侧：导航
- 中间：页面主工作区

右侧状态工作台不是所有页面的固定组成部分。

### 4.2 三栏结构只属于主页面

三栏结构只适用于 `此刻` 页面。

原因：

- 此刻页承担全局实时状态
- 此刻页需要对话流与状态工作台同时存在
- 此刻页本身就是 Echo 的主驾驶舱

`此刻` 页结构：

- 左侧导航
- 中间主对话流
- 右侧状态工作台

### 4.3 其他四页不固定保留右侧状态栏

`学习 / 行动 / 反思 / 记忆` 采用扩展主内容布局：

- 左侧导航保留
- 中间内容区扩展
- 不重复挂完整右侧状态栏
- 如有必要，只保留极少量页面内摘要模块

这样可以避免：

- 状态卡内容重复
- 页面目标被冲淡
- 五个页面看起来只是“同一页面换标题”

## 5. 页面目标

### 5.1 此刻

目标：

- 看当前状态
- 和 Echo 继续对话
- 接住下一步行动

主动作：

- 发送输入 / 继续对话

### 5.2 学习

目标：

- 查看当前学习主题
- 理解步骤推进
- 推进当前步骤

主动作：

- 推进当前步骤

### 5.3 行动

目标：

- 明确当前主任务
- 查看次级任务
- 开始或完成当前任务

主动作：

- 开始 / 完成当前任务

### 5.4 反思

目标：

- 查看简短总结
- 理解趋势和模式

主动作：

- 刷新总结

### 5.5 记忆

目标：

- 查看关键召回片段
- 理解标签与记忆分组

主动作：

- 查看 / 置顶关键记忆

## 6. 信息密度

Echo 不应依赖大段文字解释自己。

规则：

- 卡片内正文默认不超过两行
- 状态区优先使用图形化表达
- 中间对话区允许自然语言较多
- 右侧状态区必须高度可扫描
- 主层不承载长说明

适合放在主层的文字：

- 状态名称
- 任务标题
- 当前步骤
- 一句说明
- 一句反馈

不适合主层的内容：

- 长教程
- 大段引导
- 重复解释
- 冗长空状态文案

## 7. 视觉方向

Echo 应同时具备以下三种质感：

1. 苹果式秩序
2. 轻桌面产品感
3. 轻微数字纹理记忆感

视觉优先级：

1. 可读性
2. 当前最重要动作
3. 空间平衡
4. 直接状态信号
5. 安静气质
6. 纹理点缀

## 8. 当前已确认的主视觉结论

后续所有页面图和前端实现都必须遵守以下结论：

1. 使用黑、白、灰、蓝作为唯一主色系统
2. 整体遵循苹果式克制桌面布局
3. 左侧为黑色垂直导航轨
4. `此刻` 页采用三栏结构
5. `学习 / 行动 / 反思 / 记忆` 不固定保留右侧状态栏
6. 中间区域以主流程为核心
7. 保留 Current State 的心跳波形
8. 保留 Focus 的可视化进度条
9. 保留 Learning 的分段进度表现
10. 减少文字占比，提高图形表达比例

## 9. 颜色规范

只允许使用黑、白、灰、蓝体系。

### 9.1 Core

```css
--color-black: #050608;
--color-ink: #111318;
--color-graphite: #242832;
--color-gray-900: #1c1f26;
--color-gray-700: #4b5565;
--color-gray-500: #7d8594;
--color-gray-300: #c9ced8;
--color-gray-200: #e5e8ee;
--color-gray-100: #f2f4f7;
--color-white: #ffffff;
```

### 9.2 Blue Accent

```css
--color-blue: #2563eb;
--color-blue-hover: #1d4ed8;
--color-blue-soft: #dbeafe;
--color-blue-line: #93c5fd;
--color-blue-ink: #0f3d91;
```

### 9.3 State Signal

```css
--state-quiet: #7d8594;
--state-focused: #2563eb;
--state-motivated: #1d4ed8;
--state-anxious: #0f3d91;
--state-distracted: #4b5565;
--state-neutral: #9ca3af;
```

规则：

- 状态差异优先靠蓝色强度、灰度对比、频率、振幅、线型
- 不引入绿、黄、红作为主状态色
- 不引入紫、米、橙、棕等新配色主题

## 10. 字体规范

主字体：

```css
font-family:
  -apple-system,
  BlinkMacSystemFont,
  Inter,
  "SF Pro Display",
  "Segoe UI",
  "PingFang SC",
  "Microsoft YaHei",
  sans-serif;
```

等宽字体：

```css
font-family:
  "JetBrains Mono",
  "SFMono-Regular",
  Consolas,
  "Liberation Mono",
  monospace;
```

字号层级：

- App 标题：`24px / 32px`，`650`
- 页面标题：`20px / 28px`，`650`
- 模块标题：`14px / 20px`，`650`
- 正文：`14px / 22px`，`400`
- 小正文：`13px / 20px`，`400`
- 元信息：`11px / 16px`，`500`
- 按钮：`13px / 18px`，`600`

规则：

- 不随 viewport 缩放字号
- 正文字距保持 0
- 中文与英文遵循同一层级

## 11. 间距规范

采用 4px 基础与 8px 节奏体系。

```css
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 20px;
--space-6: 24px;
--space-8: 32px;
--space-10: 40px;
```

推荐值：

- 外层 padding：`16px`
- 主面板 padding：`20px`
- 标准卡片 padding：`16px`
- 紧凑卡片 padding：`12px`
- 卡片堆叠间距：`12px`
- 控件间距：`8px`

## 12. 圆角、边框、阴影

```css
--radius-sm: 4px;
--radius-md: 6px;
--radius-lg: 8px;
--border-width: 1px;
--shadow-subtle: 0 1px 2px rgba(17, 19, 24, 0.06);
--shadow-panel: 0 12px 30px rgba(17, 19, 24, 0.08);
```

规则：

- 按钮：`6px`
- 输入框：`6px`
- 面板：`8px`
- 主要依赖边框，不依赖重阴影

## 13. 图标与原生控件

- 默认图标库：`lucide-react`
- 禁止 emoji
- 优先使用原生或 headless 可访问组件
- 保留清晰 focus 状态
- 不为了“设计感”牺牲键盘可用性

## 14. 左侧导航

左侧导航保持黑、窄、稳定。

保留项：

- 此刻
- 学习
- 行动
- 反思
- 记忆
- 设置 / 同步

规则：

- 只做切换，不做复杂说明
- 激活态必须明显
- 导航图标约 `20px`
- 图标按钮约 `40px`

## 15. 此刻页结构

此刻页是 Echo 的主驾驶舱。

结构：

- 顶部状态头
- 当前状态主视觉与当前主任务摘要带
- 聊天框式主对话流
- 输入区
- 右侧状态工作台

规则：

- 对话流是绝对核心，必须保持真实交流感
- 当前状态主视觉必须简洁明确
- 当前主任务摘要要清楚，但不能压过对话流
- 输入区必须稳定、低摩擦，并支持多模态入口

### 15.1 对话流表现

此刻页中间区域必须采用聊天框形式，而不是日志面板形式。

规则：

- Echo 与用户消息以聊天消息块呈现
- 时间标注放在消息角落，弱化处理
- 对话区应像持续交流空间，而不是任务记录表
- 允许自然语言表达困惑、陪伴、澄清、安抚和推进

### 15.2 输入区表现

输入区默认按“文本 + 语音入口”设计。

建议组成：

- 文本输入框
- 语音输入按钮
- 发送按钮
- 轻量语音播放 / 说话状态提示

规则：

- 不做复杂录音控制台
- 不堆叠过多按钮
- 重点让用户感觉既能打字，也能直接和 Echo 说话

## 16. 其他四页结构

### 学习页

更像学习推进页：

- 顶部主题和总进度
- 中部步骤流
- 下部当前步骤详情或提示

### 行动页

更像任务执行页：

- 顶部主任务
- 中部任务队列
- 下部阻碍与辅助

### 反思页

更像总结页：

- 顶部今日或近期结论
- 中部趋势与模式
- 下部历史摘要

### 记忆页

更像召回页：

- 顶部记忆概览
- 中部分组与标签
- 下部关键片段

## 17. 右侧状态工作台

右侧状态工作台只作为 `此刻` 页核心特征出现。

建议保留卡片：

1. 当前状态
2. 专注
3. 下一步行动
4. 学习线
5. 成长进度
6. 记忆或反思趋势（二选一，默认弱化其一）

禁止：

- 扩展成十几个状态卡
- 变成分析后台
- 视觉上压过中间主流程

### 17.1 成长进度卡

第五张卡优先定义为“成长进度卡”，而不是纯统计卡。

它表达的重点不是 KPI，而是：

- 这周 / 这月是否持续在推进
- Echo 是否见证到了成长轨迹
- 用户是否形成了稳定的行动节奏

建议元素：

- 本周完成量
- 本月完成量
- 轻量热力格或连续性格点
- 一句里程碑式总结

规则：

- 不做重报表
- 不做大面积分析图
- 更像陪伴式成长证据，而不是绩效统计

## 18. 核心状态可视化

### 18.1 心跳波形

Current State 必须保留心跳动画。

结构：

- 状态名
- 波形
- 一句说明

状态映射：

- Quiet：低振幅、低频率、灰蓝
- Focused：中振幅、稳定频率、清晰蓝
- Motivated：较强蓝、前进节奏
- Anxious：尖峰更多、频率更高、深蓝
- Distracted：节奏不规则、对比更低
- Neutral：接近平线、弱灰

动效规则：

- 微动即可
- 循环 `1.4s` 到 `3.2s`
- 尊重 reduced motion

### 18.2 Focus 进度条

Focus 卡必须保留进度条。

结构：

- 主题
- 水平进度条
- 百分比或步骤值

### 18.3 Learning 分段进度

Learning Line 使用分段条，而不是长段说明。

规则：

- 已完成：蓝色
- 当前步骤：蓝色高亮
- 未完成：浅灰

### 18.4 Memory 网格

Memory 使用紧凑网格或密度可视化。

规则：

- 以灰和蓝表达
- 主卡只显示数量和概览

## 19. 任务化设计原则

Echo 可以借用游戏任务系统理念，但不做花哨游戏界面。

### 19.1 任务必须具备四要素

每个任务都应具备：

1. 明确标题
2. 明确当前目标
3. 明确完成条件
4. 明确进度反馈

### 19.2 主线优先

- 始终突出一个主任务
- 次任务必须弱化
- 不让多个任务同时争夺注意力

### 19.3 小步推进

- 大任务必须拆成小步骤
- 当前步骤必须清楚
- 用户不需要猜“下一步是什么”

### 19.4 轻反馈，不喧闹

允许：

- 进度推进
- 阶段完成反馈
- 下一任务解锁感
- AI 写的任务总结或任务结语

不建议首版加入：

- 复杂等级系统
- 大量勋章墙
- 金币式奖励
- 高噪音特效

### 19.5 成就系统的定位

成就是二期可扩展能力，不是首版核心主流程。

适合放在：

- 个人档案页
- 成长页
- 记忆 / 画像扩展页

成就规则必须结构化判断，成就名称和描述可以由 AI 生成润色。

### 19.6 情绪价值文案

任务提示、状态说明和成长反馈应优先采用陪伴式语气，而不是命令式语气。

例如更推荐：

- “我们先把这一步说清楚”
- “这周我们有 5 天都在向前走”
- “先别急，我们把它拆小一点”

不推荐：

- “立即完成任务”
- “你的效率偏低”
- “请按步骤执行”

## 20. 按钮与输入

### Primary

```css
background: #2563eb;
color: #ffffff;
border: 1px solid #2563eb;
border-radius: 6px;
```

### Secondary

```css
background: #ffffff;
color: #111318;
border: 1px solid #c9ced8;
border-radius: 6px;
```

### Ghost

```css
background: transparent;
color: #4b5565;
border: 1px solid transparent;
```

规则：

- 每个局部区域只有一个主按钮
- 低频操作弱化
- 复杂操作进入详情层

输入框：

- 高度：`40px` 到 `44px`
- 聚焦边框：蓝色
- 占位文字：灰色

## 21. 动画与反馈

动效必须服务理解，而不是服务炫技。

推荐：

- Hover：`120ms ease`
- 内容切换：`160ms ease`
- 常规过渡：`120ms` 到 `220ms`

反馈必须统一考虑：

- 加载态
- 空状态
- 异常态
- 同步态
- 成功反馈

规则：

- 优先 skeleton，不优先大 spinner
- 空状态一句话即可
- 错误态保持克制

## 22. 图表原则

优先 data-lite instrument，而不是正式大图表。

优先级：

1. 心跳波形
2. 进度条
3. 分段条
4. 记忆网格
5. 小趋势线
6. 仅必要时使用正式图表

允许图表库：

- Recharts
- Nivo
- Apache ECharts

## 23. 文案原则

- 优先中文标题与中文界面标签
- 使用短句
- 不说教
- 不营销
- 不过度激励
- 不使用 emoji

## 24. 缩放与适配

必须考虑 Windows 高分屏与缩放场景，尤其是：

- 2K 分辨率
- 150% 缩放
- 笔记本纵向空间不足

规则：

- 先保证完整可见
- 再保证视觉平衡
- 不能因缩放导致重叠、截断、难用

## 25. 基于本文件出图时必须遵守

1. `此刻` 页使用三栏
2. 其他四页不强制右侧状态栏
3. 保持黑白灰蓝体系
4. 保留心跳波形与 Focus 进度条
5. 减少长文本
6. 突出主流程
7. 保持轻任务化推进感
8. 不引入新风格
9. 不过度复杂化
10. 必须更像真实产品，而不是概念展示板

## 26. 禁止事项

- 不做 landing page
- 不做营销式 hero
- 不做全站统一三栏
- 不引入紫、米、橙、绿等新配色
- 不使用 emoji
- 不嵌套卡片
- 不做社交 feed 风格
