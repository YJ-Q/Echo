# Echo Style Guide

本文件定义 Echo 桌面端的统一视觉规范。  
当前基准版本采用 `此刻页 3 号方案`，内部命名为 `Breathing Minimal / 留白呼吸`。

这套风格的目标不是“做得花”，而是让 Echo 在长期使用中保持：

- 陪伴感
- 连续性
- 推进感
- 低负担
- 高耐看度

Echo 不是营销官网，不是重型任务系统，也不是炫技型 AI 面板。  
它应该像一个安静、可靠、克制、可长期相处的桌面陪伴体。

---

## 1. 风格总纲

### 1.1 关键词

- 留白
- 呼吸感
- 克制
- 原生
- 安静
- 连续陪伴
- 轻推进

### 1.2 总体气质

整体视觉参考 Apple 系设计语言，但不照搬。  
重点不是“像苹果”，而是具备以下品质：

- 信息层级清晰
- 布局秩序稳定
- 留白自然
- 视觉负担低
- 动效克制
- 长时间观看不疲劳

### 1.3 页面情绪

Echo 默认情绪应是：

- 温和
- 稳定
- 不催促
- 不冷漠

允许存在推进感，但推进感必须来自：

- 清晰的下一步
- 减少犹豫的文案
- 明确的状态反馈

而不是来自：

- 强刺激色块
- 夸张按钮
- 大量高对比装饰
- 游戏化夸张反馈

---

## 2. 适用范围

本规范适用于：

- `此刻`
- `学习`
- `行动`
- `反思`
- `记忆`

优先影响：

- 布局
- 字体
- 间距
- 颜色
- 卡片
- 输入区
- 按钮
- 状态组件
- 动效

---

## 3. 布局原则

### 3.1 全局框架

桌面端采用稳定的应用级框架：

- 左侧：导航栏
- 中部：主内容区
- 右侧：辅助状态区

其中：

- `此刻页` 使用三栏布局
- `学习 / 行动 / 反思 / 记忆` 可以隐藏右栏，扩大中区

### 3.2 留白原则

留白必须被视为功能性空间，不是“空出来没填”。

留白的作用：

- 帮助用户缓冲情绪
- 降低视觉噪音
- 突出当前主线
- 让单个动作更明确

禁止：

- 为了“看起来满”而补无意义内容
- 在每个区块塞统计、图标、装饰标签
- 卡片层层嵌套导致页面拥堵

### 3.3 聚焦原则

任意时刻页面只能有一个主动作。

例如：

- 此刻页：发送输入
- 学习页：设为进行中 / 标记完成
- 行动页：设为进行中 / 完成任务

同屏不允许多个同等级 CTA 抢注意力。

## 4. 字体规范

### 4.1 字体策略

整体字体应优先选择原生、清洁、易读的字体系统。  
不追求强烈个性字体，不使用花哨展示字体作为主文字体。

推荐字体栈：

```css
font-family:
  "SF Pro Display",
  "SF Pro Text",
  -apple-system,
  BlinkMacSystemFont,
  "Segoe UI",
  "PingFang SC",
  "Hiragino Sans GB",
  "Microsoft YaHei",
  sans-serif;
```

等宽字体仅用于：

- 时间
- 状态码
- 小型系统标签
- 调试信息

推荐：

```css
font-family:
  "SF Mono",
  "JetBrains Mono",
  "Cascadia Code",
  Consolas,
  monospace;
```

### 4.2 字号层级

```css
--font-display: 32px;
--font-h1: 24px;
--font-h2: 18px;
--font-h3: 15px;
--font-body: 14px;
--font-small: 13px;
--font-micro: 11px;
```

推荐使用场景：

- `32px`：页面核心情绪词、主视觉标题
- `24px`：页面标题
- `18px`：卡片标题 / 模块标题
- `15px`：次级重点信息
- `14px`：正文、输入、按钮主文案
- `13px`：辅助说明
- `11px`：元信息、标签、时间、系统状态

### 4.3 字重

```css
--weight-regular: 400;
--weight-medium: 500;
--weight-semibold: 600;
--weight-bold: 700;
```

使用建议：

- 正文：`400`
- 普通按钮 / 标签：`500`
- 卡片标题 / 页面标题：`600`
- 情绪词 / 核心焦点：`700`

### 4.4 行高

```css
--line-tight: 1.2;
--line-normal: 1.5;
--line-relaxed: 1.7;
```

使用建议：

- 标题：`1.2`
- 常规正文：`1.5`
- 较长说明：`1.7`

### 4.5 文案表现

文本风格应：

- 句子短
- 语气稳
- 避免命令感
- 优先中文

不使用：

- emoji
- 夸张感叹号堆叠
- 过度营销化文案

## 5. 颜色规范

### 5.1 颜色原则

整体采用 `黑 / 白 / 灰 / 蓝` 体系。  
3 号方案的核心是“亮背景 + 弱对比 + 单一蓝色强调”。

颜色的职责分配：

- 白与浅灰：承载空间和呼吸感
- 深灰与黑：承载信息和秩序
- 蓝色：承载状态强调和推进感

禁止：

- 引入紫色主基调
- 大面积高饱和渐变
- 复杂多彩状态系统
- 红绿黄作为主视觉核心色

### 5.2 基础色板

```css
--color-white: #ffffff;
--color-bg: #f5f7fb;
--color-bg-soft: #eef3f8;
--color-surface: rgba(255, 255, 255, 0.82);
--color-surface-strong: rgba(255, 255, 255, 0.94);

--color-ink: #111827;
--color-ink-soft: #1f2937;
--color-text: #18212f;
--color-text-secondary: #5f6b7c;
--color-text-tertiary: #8a94a6;

--color-line: rgba(24, 33, 47, 0.08);
--color-line-strong: rgba(24, 33, 47, 0.14);

--color-blue: #2463eb;
--color-blue-hover: #1f57d0;
--color-blue-deep: #1544b2;
--color-blue-soft: rgba(36, 99, 235, 0.1);
--color-blue-surface: rgba(36, 99, 235, 0.06);
```

### 5.3 功能色使用原则

蓝色仅用于：

- 当前激活导航
- 主按钮
- 关键进度
- 当前状态强调
- 重点可操作路径

灰色用于：

- 辅助文案
- 边框
- 弱标签
- 未激活状态

深色用于：

- 页面主标题
- 关键信息
- 核心数字

### 5.4 背景策略

默认背景不是纯白，而是轻微带冷感的浅灰白。

推荐背景组合：

```css
background:
  radial-gradient(circle at top left, rgba(36, 99, 235, 0.08), transparent 26%),
  radial-gradient(circle at bottom right, rgba(36, 99, 235, 0.06), transparent 22%),
  linear-gradient(180deg, #f8fafd, #f3f6fa);
```

目标：

- 有环境感
- 但不抢内容
- 不形成“科技海报感”

## 6. 间距规范

### 6.1 基础 spacing scale

统一采用 4px 基准：

```css
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 20px;
--space-6: 24px;
--space-7: 28px;
--space-8: 32px;
--space-10: 40px;
--space-12: 48px;
```

### 6.2 使用建议

- 页面外边距：`24px - 36px`
- 卡片内边距：`16px - 20px`
- 大区块之间：`24px`
- 模块内元素之间：`8px - 12px`
- 输入区按钮间距：`8px`
- 标签组间距：`8px`

### 6.3 视觉密度

Echo 默认使用 `舒适密度`，不走信息压缩路线。

规范：

- 单卡片内部不堆太多层级
- 大区块之间保持明显呼吸空间
- 长文本上下必须留足空间
- 快捷操作保持短小而不拥挤

---

## 7. 圆角、边框、阴影

### 7.1 圆角

3 号方案整体圆角应偏柔和，但不能过甜。

```css
--radius-xs: 10px;
--radius-sm: 12px;
--radius-md: 16px;
--radius-lg: 20px;
--radius-xl: 24px;
--radius-2xl: 28px;
```

建议：

- 按钮：`12px`
- 输入框：`14px - 16px`
- 卡片：`18px - 24px`
- 大容器：`24px - 28px`

### 7.2 边框

边框比阴影更重要。  
Echo 的层次优先通过弱描边建立，而不是重阴影。

```css
--border-subtle: 1px solid rgba(24, 33, 47, 0.08);
--border-strong: 1px solid rgba(24, 33, 47, 0.14);
```

### 7.3 阴影

阴影必须轻。

```css
--shadow-xs: 0 2px 8px rgba(15, 23, 42, 0.04);
--shadow-sm: 0 8px 24px rgba(15, 23, 42, 0.06);
--shadow-md: 0 18px 50px rgba(15, 23, 42, 0.08);
```

禁止：

- 浓重悬浮阴影
- 明显“卡片漂浮”感过强
- 暗色厚投影破坏安静感

## 8. 组件风格

### 8.1 导航栏

导航栏应：

- 安静
- 固定
- 清晰标识当前页

当前页可使用蓝色高亮，但高亮面积不要过大。  
非当前项保持弱对比。

### 8.2 卡片

卡片是 Echo 的核心容器。

卡片风格要求：

- 以浅底为主
- 弱边框
- 圆角柔和
- 内容层级少而清楚

卡片内部通常只允许：

- 一个标题
- 一段短说明
- 一个主状态或一个主动作

### 8.3 对话气泡

对话是 Echo 的核心，不应做成普通 IM，也不应做成日志板。

要求：

- Echo 与用户消息有清晰区分
- Echo 气泡更柔和
- 用户气泡更简洁
- 时间信息弱化
- 单条气泡不宜过宽

建议宽度：

- 最大宽度 `72% - 84%`

### 8.4 输入区

输入区必须稳定、低摩擦。

要求：

- 保持单行优先
- 形态完整
- 主按钮明确
- 辅助操作不喧宾夺主

输入框视觉建议：

- 浅色实体底
- 弱描边
- 聚焦时蓝色边框增强

### 8.5 按钮

按钮分三层：

- Primary
- Secondary
- Ghost

Primary：

```css
background: linear-gradient(180deg, #3977ff, #2463eb);
color: #ffffff;
border: 1px solid #2463eb;
```

Secondary：

```css
background: rgba(255, 255, 255, 0.88);
color: #18212f;
border: 1px solid rgba(24, 33, 47, 0.1);
```

Ghost：

```css
background: transparent;
color: #5f6b7c;
border: 1px solid transparent;
```

### 8.6 标签与状态 Chips

标签必须轻。

建议：

- 高度 `22px - 26px`
- 字号 `11px - 12px`
- 弱色底
- 小圆角或胶囊形

不要把标签做成彩色糖豆风格。

---

## 9. 状态与图形化表达

### 9.1 图形原则

Echo 可以使用轻图形化状态组件，但必须服务理解，不服务炫技。

允许：

- 进度条
- 分段进度
- 圆环状态
- 波形 / 节律线
- 极简趋势线

不建议：

- 复杂 dashboard 图表
- 多色大图
- 高频动画图形

### 9.2 3 号方案推荐图形

优先使用：

- 细圆环
- 极简分段条
- 呼吸型状态面
- 轻量波形

视觉特征：

- 线条细
- 蓝色低饱和
- 保持留白
- 不把图形做满

---

## 10. 动效规范

### 10.1 动效原则

动效必须：

- 短
- 轻
- 稳
- 有意义

服务场景：

- 页面切换
- 卡片进入
- 消息出现
- 状态变化
- 加载反馈

### 10.2 推荐时长

```css
--motion-fast: 120ms;
--motion-base: 180ms;
--motion-slow: 260ms;
```

推荐 easing：

```css
cubic-bezier(0.22, 1, 0.36, 1)
```

### 10.3 动效风格

推荐：

- 淡入
- 轻位移
- 柔和展开
- 进度平滑增长

避免：

- 弹跳
- 夸张缩放
- 高频闪烁
- 粒子、辉光、霓虹效果

### 10.4 加载反馈

优先：

- 骨架屏
- 面积稳定的占位块

次选：

- 小型 spinner

不建议：

- 全屏转圈
- 长时间无反馈

---

## 11. 内容与文案语气

### 11.1 语气原则

Echo 文案应像一个稳定、聪明、温和的陪伴者。

关键词：

- 温和
- 具体
- 不评判
- 不命令
- 不说教

### 11.2 推荐表达

- 我们先把这一步说清楚
- 先不用全部想完
- 你现在可以只做这一小步
- 这已经比刚才更清楚了

### 11.3 避免表达

- 立即完成
- 你应该
- 你的效率太低
- 请严格执行

---

## 12. 响应式与桌面适配

Echo 优先服务桌面端，尤其是：

- 1440px 宽度以上桌面
- 13-16 寸笔记本
- Windows 高缩放环境

必须考虑：

- 150% 缩放
- 竖向空间不足
- 长文本折行

原则：

- 先保证可读
- 再保证层级
- 最后保证装饰完整

---

## 13. 禁止事项

- 不使用 emoji
- 不引入紫色主视觉
- 不做营销 landing page 风格 hero
- 不做赛博霓虹风
- 不做信息爆炸 dashboard
- 不堆砌图标
- 不使用重玻璃、重发光、重阴影
- 不把页面塞满假数据
- 不以装饰替代结构

---

## 14. 此刻页落地要点

基于 3 号方案，`此刻页` 必须满足：

1. 第一眼是安静的，不是兴奋的
2. 页面主标题区域有足够留白
3. 对话流仍然是核心，不被状态卡压过
4. 右侧状态区数量克制，优先显示真正有用的信息
5. 蓝色只用于真正值得注意的地方
6. 输入区必须稳定且易于继续表达
7. 整个页面让用户感觉“我可以继续”，而不是“我又被管理了”

---

## 15. 推荐设计结论

如果后续实现只记住一件事，请记住：

`Echo 的 3 号方案不是极简展示页，而是一个有呼吸感的陪伴型工作界面。`

它的审美核心不是“少”，而是：

- 少而不断线
- 轻但不空
- 稳但不冷
- 推进但不催促
