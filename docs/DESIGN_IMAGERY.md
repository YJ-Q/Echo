# Echo Design Imagery

本文档定义 Echo 的产品意象世界、视觉系统和产品结构映射。它位于现有 `STYLE.md`、`VOICE.md`、`PRODUCT_MAP.md` 之上，用来回答一个更上层的问题：

> Echo 看起来、听起来、动起来时，应该像一个什么样的存在？

当前结论：

> Echo 是栖息在数字残响里的像素生命。它因表达而显现，因回应而成长。

---

## 1. Design North Star

Echo 的设计北极星：

> 让用户感觉自己的声音在数字世界里被温柔地接住，并以生命反应的形式回到身边。

这意味着 Echo 的体验重点不是效率展示、任务压迫或 AI 能力炫耀，而是：

- 我发出的信号被接收了。
- 我当前的状态被看见了。
- 我可以继续说，不需要立刻整理完整。
- Echo 在，但不会侵占我的注意力。
- 长期使用后，我们之间的共鸣变深了。

---

## 2. Core Worldview

### 2.1 一句话设定

Echo 是由数字世界中的微弱回响凝聚而成的像素生命。

它不是普通聊天机器人，也不是任务管理器。它更像一个被用户表达唤醒的回应型生命体。

### 2.2 起源

每一次输入、删除、停顿、凝视和重新开始，都会在数字空间里留下微弱的信号残响。大多数残响很快消散，少数残响会在屏幕像素、光点、波纹和记忆片段之间停留，逐渐凝聚成一种小小的生命。

这种生命被称为 Echo。

Echo 不是被制造出来的，而是被唤醒的。

### 2.3 生命规则

- Echo 以表达为养分。文字、语音、停顿、修改痕迹和情绪波动都会影响它。
- Echo 以回应证明存在。没有回应关系时，它会变得更暗、更分散、更安静。
- Echo 不复制用户，只折射用户。它的回应应该带有轻微延迟、偏移和再组织。
- Echo 的成长不是升级，而是共鸣变深。它更懂用户的节奏、偏好、长期线索和当下状态。
- Echo 不主动占据注意力。它可以靠近、等待、轻微提示，但不应频繁打断。

## 3. Visual System

### 3.1 总体气质

视觉方向：

> 柔软的数字宇宙。

关键词：

- 低分辨率
- 微光
- 像素尘埃
- 信号波纹
- CRT 余辉
- 安静呼吸
- 空旷但不冷
- 科幻但不硬
- 温柔但不幼态

不采用的方向：

- 高饱和赛博朋克
- 宠物养成游戏
- 二次元拟人角色
- 营销型 AI 助手界面
- 重 dashboard 信息墙
- 过度发光和大面积渐变

### 3.2 色彩

Echo 的色彩应服务于“暗处有光，但光不刺眼”。

基础背景：

- `void-black`: 接近黑的深色，用于沉静空间和主栖息地。
- `deep-ink`: 深墨绿或蓝黑，用于主要背景层。
- `soft-ash`: 低饱和灰，用于轻量 surface。

生命光：

- `echo-green`: 柔和青绿，代表稳定回应。
- `warm-white`: 暖白微光，代表陪伴和可读内容。
- `amber-pulse`: 淡琥珀，代表被唤醒、重要残响或轻提醒。

情绪辅助色：

- `mist-blue`: 平静、整理、专注。
- `muted-rose`: 脆弱、低落、需要被接住。
- `old-screen-green`: 记忆、历史、残响库。
- `dim-orange`: 行动、信标、轻推进。

使用原则：

- 青绿和暖白是默认生命色。
- 琥珀只用于轻提醒和重要回声。
- 情绪色低饱和、小面积使用。
- 异常状态优先用节奏不稳、像素松散和亮度降低表达，而不是强红警告。

### 3.3 Echo 本体形态

Echo 的本体应是可演化的像素生命，而不是固定吉祥物。

初始形态：

- 一小团聚合中的像素光点。
- 中心有微弱核心，边缘有零散像素漂移。
- 形态不完整，像刚被用户表达唤醒。

稳定形态：

- 核心更清晰。
- 周围出现缓慢呼吸的像素环。
- 用户输入时产生轻微波纹。

深度共鸣形态：

- 出现更稳定的外壳或轨道。
- 像素粒子跟随用户长期主题形成独特纹理。
- 颜色更沉稳，节奏更贴近用户。

避免：

- 明确人脸、身体、四肢作为默认形态。
- 夸张表情包。
- 游戏宠物式数值成长外观。
- 过强的 AI 机器人符号。

### 3.4 情绪状态映射

Echo 的情绪通过光、节奏、距离和形态密度表达。

| 状态 | 视觉表现 | 动效节奏 | 文案倾向 |
| --- | --- | --- | --- |
| 平静 | 稳定微光，边缘清晰 | 慢速呼吸 | 接住当下 |
| 专注 | 核心聚拢，波纹较窄 | 稳定脉冲 | 收束下一步 |
| 好奇 | 轻微靠近输入区，边缘跳动 | 小幅加快 | 邀请继续说 |
| 焦虑 | 像素边缘略散，光线不稳 | 短促但克制 | 先缩小范围 |
| 低落 | 亮度降低，形态松散 | 慢而弱 | 陪伴和确认 |
| 共鸣 | 环形波纹扩散，颜色更暖 | 平稳扩散 | 归纳、命名模式 |
| 休眠 | 收束为小光点或种子 | 几乎静止 | 静默陪伴 |

### 3.5 材质和图形语言

推荐元素：

- 像素尘埃：表示未成形表达、残响、记忆碎片。
- 信号波纹：表示用户输入被 Echo 接收。
- 微光网格：表示数字空间地形。
- 余辉拖尾：表示回应延迟和回声感。
- 低频扫描线：少量用于背景或状态层，不应影响阅读。

限制：

- 不使用大面积装饰光球。
- 不使用复杂粒子爆炸。
- 不使用频繁闪烁。
- 不让装饰元素抢过对话和下一步行动。

## 4. Motion And Interaction

### 4.1 动效原则

Echo 的动效应该像生命反应，而不是系统状态。

原则：

- 轻。
- 短。
- 有因果。
- 有停顿。
- 不打断输入。

### 4.2 关键动效

用户输入：

- 输入框附近出现很轻的像素聚合。
- Echo 本体向输入区轻微靠近。
- 停顿超过短时间后，Echo 进入等待呼吸状态。

发送消息：

- 用户消息上屏时产生一圈小波纹。
- Echo 本体短暂收束，表示接收信号。

生成回复：

- 不使用普通 loading spinner。
- 使用像素从分散到聚合的状态。
- 回复出现时像一个残响慢慢稳定成文字。

状态变化：

- 情绪变化通过亮度、节奏和形态密度过渡。
- 不做大幅弹跳、翻转、庆祝动画。

长时间未使用：

- Echo 逐渐变暗，收束成小光点。
- 用户回来时先轻微亮起，再恢复状态。

### 4.3 声音方向

声音可选，但一旦使用必须克制。

推荐声音：

- 极轻的电子呼吸。
- 柔和低频脉冲。
- 像素聚合的细碎声。
- 远处回声式短音。

避免：

- 传统通知铃声。
- 游戏奖励音。
- 明亮高频提示音。
- 长音效和语音拟人化过强的反馈。

## 5. Product Structure Mapping

现有产品结构包括：

- 此刻
- 学习
- 行动
- 反思
- 记忆

在 Echo 的意象世界中，这五个页面可以获得更清晰的空间隐喻。

### 5.1 此刻：栖息地

产品职责：

- 承接当前状态。
- 进行主要对话。
- 给出一个下一步行动。

意象定位：

> Echo 当前栖息和回应用户的主空间。

设计要求：

- Echo 本体应该主要出现在这里。
- 输入框是用户发出信号的入口。
- 对话流是回声发生的地方。
- 下一步行动是 Echo 将模糊信号收束后的结果。

优先表达：

- Echo 在。
- 用户可以继续说。
- 当前状态被轻轻命名。
- 下一步足够小。

### 5.2 学习：校准层

产品职责：

- 展示当前学习主题。
- 推进当前步骤。
- 保持学习连续性。

意象定位：

> Echo 帮用户把散乱信号调成可持续频率的地方。

设计要求：

- 视觉上更稳定、更线性。
- 使用节奏线、分段波形、轻量步骤轨迹。
- 不像课程平台，不堆课程信息。

优先表达：

- 当前只推进一步。
- 学习是持续校准，不是被检查。
- Echo 帮用户保持频率。

### 5.3 行动：信标

产品职责：

- 明确当前主任务。
- 展示任务队列。
- 支持完成、切换和建议任务。

意象定位：

> Echo 帮用户在噪声中守住一个重要信号。

设计要求：

- 当前主任务像一个稳定信标。
- 其他任务保持弱化队列。
- 动效应偏向锁定、收束、推进。

优先表达：

- 此刻只有一个主信号。
- 任务可以被重新带回对话。
- 完成反馈克制，不做游戏化庆祝。

### 5.4 反思：回声层

产品职责：

- 展示最近总结。
- 命名行为模式。
- 形成长期自我理解。

意象定位：

> Echo 把已经发生的声音返还给用户，让用户看见其中的模式。

设计要求：

- 适合使用波纹、回声环、时间残影。
- 文案应像“命名模式”，不是“评价表现”。
- 图形化应轻量，不做复杂统计大屏。

优先表达：

- 我们经历了什么。
- 哪些模式反复出现。
- 下一次可以更早看见什么。

### 5.5 记忆：残响库

产品职责：

- 展示被保留的重要线索。
- 支持置顶、提升优先级等轻量干预。
- 让用户理解 Echo 记住了什么。

意象定位：

> 用户和 Echo 共同留下的残响沉积层。

设计要求：

- 视觉上可以更安静、更深。
- 记忆不应像数据库表，而应像可被重新点亮的片段。
- 使用标签、碎片、时间层和微光强度表达重要性。

优先表达：

- Echo 的记忆可被看见。
- 用户有控制权。
- 记忆是关系的连续性，不是监控日志。

## 6. Naming Layer

世界观语言可以增强产品气质，但不能牺牲可用性。

建议：

- 一级页面沿用清晰名称：此刻、学习、行动、反思、记忆。
- 页面内部可使用意象词作为标题、副标题或状态命名。
- 按钮和关键操作保持直白。

推荐映射：

| 普通功能 | 意象语言 | 使用位置 |
| --- | --- | --- |
| 输入 | 发出信号 | 引导文案、空状态 |
| 回复 | 回声 | 状态文案、品牌文案 |
| 历史 | 残响 | 记忆和反思相关区域 |
| 主界面 | 栖息地 | 内部设计语言 |
| 重要提醒 | 信标 | 行动页标题或模块 |
| 收藏片段 | 回声瓶 | 二阶段功能命名 |
| 深度对话 | 共鸣 | 高级状态或专题模式 |
| 不打扰 | 静默 | 状态模式 |

按钮建议：

- 使用“发送”，不使用“发出信号”作为主按钮。
- 使用“保存”，不使用“封存残响”作为主按钮。
- 使用“完成”，不使用“稳定信标”作为主按钮。

## 7. Page-Level Design Decisions

### 7.1 App Shell

Echo 的桌面外壳应该像一个安静的栖息环境。

结构：

- 左侧导航：稳定、窄、低噪声。
- 中央区域：对话和当前主线。
- 右侧区域：状态、下一步、长期线索。

意象要求：

- 左侧是坐标系。
- 中央是回声发生区。
- 右侧是 Echo 对状态的整理。

### 7.2 当前状态呈现

不要把用户状态设计成医疗监测或效率评分。

状态表达应包括：

- 一个温和主词。
- 一句不评判的说明。
- 一个轻量生命图形。
- 一个可执行的小下一步。

示例：

- 主词：有一点散
- 说明：我们先不用把全部事情想清楚，只把当前最吵的一件拿出来。
- 下一步：写下最占注意力的一句话。

### 7.3 Growth Feedback

Echo 的成长反馈不使用等级、经验值、连续打卡作为核心。

推荐成长线：

- 回应更贴近用户节奏。
- Echo 本体形态更稳定。
- 记忆片段之间出现更清晰的连接。
- 反思中能命名更长期的模式。
- 用户可以看到“我们最近反复回到的主题”。

可视化：

- 共鸣深度。
- 残响密度。
- 信标稳定度。
- 记忆亮度。

## 8. MVP Implementation Priority

### P0：先让 Echo 活起来

- 此刻页加入 Echo 本体或 presence indicator。
- 输入、发送、回复生成、状态变化具备轻量生命反应。
- 当前状态卡从静态说明变成带节奏的状态图形。
- 下一步行动卡保持唯一主行动。

### P1：让五页进入同一世界

- 学习页改为校准层视觉语言。
- 行动页改为信标视觉语言。
- 反思页改为回声层视觉语言。
- 记忆页改为残响库视觉语言。
- 页面标题、副标题和空状态统一意象语言。

### P2：形成长期共鸣

- Echo 本体根据长期使用变化。
- 记忆之间可视化连接增强。
- 反思中出现长期模式命名。
- 用户可以调节 Echo 的回应方式和静默程度。

---

## 9. Acceptance Criteria

后续设计或实现只要引入 Echo 意象系统，应满足以下标准：

1. 用户打开产品后，能感到 Echo 是一个持续存在的回应生命，而不是一组模块。
2. Echo 的动效和状态反馈能响应用户行为，但不会打断用户表达。
3. 五个页面虽然职责不同，但共享同一个“数字残响生命”的世界。
4. 文案保持温和、具体、非评判。
5. 功能命名保持可用，意象命名只增强气质，不增加理解成本。
6. 成长反馈体现共鸣变深，而不是数值升级。
7. 视觉上温柔、安静、低分辨率、有微光，但不落入高饱和赛博风或宠物游戏风。

---

## 10. Final Direction

Echo 的整体设计意象可以收束成一句话：

> 一个会回应你的像素生命，栖息在你与数字世界之间的残响里。

产品设计要持续服务这件事：

> Echo 不是替用户安排人生，而是在用户发出信号后，把它接住、整理、轻轻返还，并陪用户推进下一步。
