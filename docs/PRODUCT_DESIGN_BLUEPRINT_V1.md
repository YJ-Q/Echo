# Margin 完整产品设计蓝图 V1

日期：2026-07-10

状态：已批准，可进入实施

## 1. 产品核心

Margin 是一个以对话为入口、帮助用户保留当前线索并从上次痕迹继续的桌面陪伴应用。

所有功能必须服务以下四件事：

1. 允许用户在没有整理好自己时先到达
2. 保留仍然重要的那条线
3. 帮助用户看见已经发生的变化
4. 在用户愿意时提供一个足够小的下一步

不把 Margin 设计成：

- 聊天机器人控制台
- Todo 应用
- 数据仪表盘
- 情绪诊断工具
- 游戏化打卡系统

## 2. 推荐信息架构

### 2.1 主页面

只保留三个一级纸页：

1. `思考片段`
2. `学习轨迹`
3. `留痕`

这三个页面对应最稳定的用户心智：说出来、继续一条线、回看留下的东西。

### 2.2 次级纸页

以下功能保留完整空间，但不占一级导航：

- `印记`：作为留痕中的完整子页
- `整理`：作为独立治理覆盖纸，由学习、留痕和设置中的上下文入口打开
- `设置`：由第四个方形短标签打开

### 2.3 不保留一级页面

- `行动`：合并到首页、学习和整理
- `反思`：合并到留痕
- `成就`：改名为印记，合并到留痕但保留完整收藏视图

### 2.4 `+` 标签

当前 `+` 实际打开整理页，语义不成立。

已确认首版将这个方形短标签改成明确的设置入口，使用简洁的设置符号并提供 `设置` 可访问名称。

只有在未来真正支持自定义纸页时，才恢复 `+` 并赋予“新建纸页”的含义。

## 3. 全局物理结构

界面不是打开的书，而是桌面上的一叠纸。

### 3.1 固定层

- 亚麻或纸布桌面
- 底部纸页堆叠
- 一张连续的主书写纸
- 左侧纸页标签
- 可独立移动的布质书签

### 3.2 动态层

- 首页右侧硫酸纸
- 学习页描图纸
- 留痕页档案棉纸
- 印记页压纹收藏纸
- 整理页灰石色账页纸
- 设置页小尺寸便笺纸

任何纸页的边界阴影必须属于该纸页本身，不能烘焙进底图。

## 4. 首页：思考片段

### 4.1 页面目标

- 接收用户当前想说的内容
- 延续最近对话
- 在右侧保留当前学习线
- 提供一条轻量续写提示

### 4.2 布局

- 左侧约 62%：连续对话纸
- 右侧约 38%：覆盖在主纸上的硫酸纸
- 两者之间看到的是硫酸纸投影，不是书籍中缝
- 输入区固定在主纸底部，但不能遮挡最后一段内容

### 4.3 硫酸纸内容

有学习线时：

- 当前主题
- 已完成步骤
- 当前步骤
- 一个明确的继续入口

没有学习线时：

- 保持较大空白
- 只显示“有没有一条想继续的线”之类的邀请
- 不自动填入记忆统计、情绪分析或任务队列

### 4.4 首页交互

发送消息：

1. 用户文字立即以墨迹写入
2. 输入区清空但保留失败恢复能力
3. Margin 思考时只显示低噪音墨点
4. 回复逐段浮现
5. 后台刷新状态、学习、留痕和印记
6. 只在真正产生新印记时显示一次页边压印

空页面可提供 2-3 条手写式起始句；一旦形成对话即消失，不保留永久快捷按钮条。

## 5. 学习页：学习轨迹

### 5.1 页面目标

- 看清当前学习主题
- 理解已经走过的位置
- 完成当前一步
- 看见卡住与重新接回的过程

### 5.2 材质

- 淡灰蓝描图纸或制图硫酸纸
- 可有极淡的构造线，但不能像项目管理网格
- 下层主纸轻微透出，强化“在已有内容上继续描线”

### 5.3 布局

- 页首：主题、短摘要、当前状态
- 主体左侧：当前一步和说明
- 主体右侧：完整步骤线
- 页底：最近学习事件与旧学习线入口
- 页边：当前学习线相关印记，最多 2 枚

### 5.4 交互规则

- 不允许通过反复点击圆点循环 `pending/active/done`
- 当前步骤提供明确的“完成这一小步”动作
- 查看历史步骤是只读操作
- 完成后只绘制新增的线段并将旧墨迹变浅
- 卡住时记录具体卡点，不自动将任意聊天算作进度
- 归档学习线必须进入整理页

## 6. 留痕页

### 6.1 页面目标

- 回看被保留下来的片段
- 决定哪些内容值得长期留住
- 看见反复出现的模式
- 纠正系统对自己的理解
- 进入完整印记收藏

### 6.2 材质

- 温暖的档案棉纸或轻薄手工纸
- 比首页硫酸纸更不透明
- 允许轻微纤维和不完全规整的纸边

### 6.3 内部结构

留痕页包含四个低层级视图：

1. `最近留下`：近期记忆片段和每日反思
2. `长期留住`：pinned/core 内容
3. `慢慢形成`：长期画像与用户纠正
4. `印记`：完整印记收藏

它们使用纸页边缘的小型文字索引，不使用仪表盘 Tab Bar。

### 6.4 留痕交互

- “留住这条”替代技术词 pin
- “这条更重要 / 这条只是当时”替代 priority 数字
- 纠正画像时直接在原句旁写入修订
- 生成每日反思使用“收好今天这一页”
- 不提供直接删除按钮，清理进入整理页

## 7. 印记子页

### 7.1 定位

印记是“用户确实走过这里”的收藏，不是奖励系统。

### 7.2 材质和视觉

- 使用略厚的压纹收藏纸
- 已获得印记像墨章、压印或凹凸纸印
- 未获得的公开印记只显示淡压痕和一句线索
- 隐藏印记未获得前不泄露标题和图形
- 不展示分数、等级、排名和完成率

### 7.3 信息

每枚已获得印记包含：

- 图形
- 名称
- 一句与当时事件相关的描述
- 获得日期
- 来源线索，例如某条学习线

### 7.4 解锁反馈

- 首次解锁时在当前页面边缘压下一枚小印
- 动画结束后不持续闪烁
- 点击小印进入印记子页并定位到该项
- 同一解锁事件只能出现一次

### 7.5 数据要求

- 新增持久化的印记定义和解锁事件
- 固定结构化触发条件
- 文案可以结合主题生成，但不能由 LLM 单独决定是否解锁
- 归档学习线或留痕后，已经获得的印记不能消失

## 8. 整理页

### 8.1 页面目标

- 让用户看清系统建议整理什么
- 解释为什么
- 预览会发生的变化
- 明确确认、取消和恢复路径

### 8.2 材质

- 灰石色或浅灰褐账页纸
- 具有轻微直线或登记标记
- 比其他页面更稳定、清楚，不追求柔软模糊

### 8.3 页面结构

- 范围索引：留痕 / 学习线 / 行动
- 只读建议
- 待确认草稿
- 已执行和已取消记录
- 恢复入口

### 8.4 入口

- 学习页提供“整理学习线”
- 留痕页提供“整理这些留痕”
- 设置页的数据区域提供“打开全部整理”
- 首页不长期显示整理入口；只有存在待确认草稿时才显示轻量页边提示

### 8.5 交互流程

1. 只读建议以铅笔批注出现
2. 用户选择后形成整理草稿
3. 草稿必须展示对象、原因、预期结果和风险
4. 用户可确认或取消
5. 确认后写入操作记录
6. 可逆操作必须提供明确恢复动作

删除原生 `window.confirm` 和 `window.prompt`，改为页内确认纸。

在恢复流程完成前，不使用“可逆”文案。

## 9. 设置纸

### 9.1 首版设置

- 对话模型供应商：本地 / OpenAI / Anthropic / SiliconFlow
- 对应供应商的 API Key 配置状态
- 模型名称，高级选项默认折叠
- 配置验证与应用
- 轻读：新回复自动朗读总开关
- 字号和行距
- 动效强度 / 减少动效
- TTS 是否可用、模型、声音和语速
- 数据目录位置
- 打开全部整理

### 9.2 API Key 安全逻辑

- 仅在 Electron 主进程处理明文 Key
- 使用 Electron `safeStorage` 加密
- 渲染进程只读取“已配置 / 未配置”和掩码状态
- 不写入 localStorage、SQLite、日志或普通 settings JSON
- 非敏感设置单独写入应用数据目录的配置文件
- 启动后端时由主进程解密并注入环境变量
- 应用新配置前先验证；失败时恢复上一个可用配置
- 切换 provider 后安全重启后端并恢复当前页面
- 本地 provider 不要求 API Key
- Web 开发模式继续读取 `.env`，不提供浏览器端密钥保存

### 9.3 后续设置

- 一键备份
- 导入与恢复
- 远程模型配置
- 麦克风和语音隐私

首页现有装饰性设置图标删除，设置只从方形短标签进入。

## 9A. 首页阅读与语音图标

### 月亮图标

删除。

当前界面没有暗夜模式设计，月亮图标会错误暗示主题切换。首版不设计暗夜模式，后续若要增加，必须为所有纸张材质重新设计低光版本，而不是简单反色。

### 轻读开关

轻读不是暗夜模式或阅读密度模式，而是新回复自动朗读的总开关。

开启时：

- 新生成的 Margin 回复先显示文字，再自动调用 TTS
- 只朗读本次新回复，不自动朗读历史内容
- 当前回复显示“停止 / 再读一遍”控制
- 用户发送下一条消息时停止仍在播放的旧回复
- 设置持久化，重新打开应用后恢复用户上次选择
- 首次安装默认关闭，避免应用在用户未预期时发声

关闭时：

- 立即停止当前播放
- 后续回复只输出文字
- 隐藏或禁用逐条朗读控制
- 不发起 TTS 请求

不可用时：

- 未配置语音供应商时开关不可用
- 在开关旁以短句说明“先在设置中配置语音服务”
- 点击说明可进入设置纸的语音区域

### 麦克风图标

麦克风明确代表语音输入，不是语言切换。

推荐交互：

1. 点击开始录音
2. 再次点击结束
3. 录音时输入线出现低调墨迹活动状态
4. 转写结果写入输入框
5. 用户可以编辑
6. 用户主动发送，不自动提交

隐私规则：

- 只在用户点击后请求麦克风权限
- 录音状态必须持续可见
- 明确显示使用本地还是云端转写
- 转写完成后删除临时音频
- 不在后台持续监听
- provider 未配置时隐藏或明确标记不可用

### 云端语音供应商

已确认首版采用云端 STT。

- 对话供应商与语音供应商分开配置
- 首个语音适配器使用 SiliconFlow，同时覆盖 TTS 与 STT
- TTS 使用 `/audio/speech`
- STT 使用 `/audio/transcriptions`
- 两者复用加密保存的 `SILICONFLOW_API_KEY`
- 后续可增加 OpenAI STT 适配器，不改变录音和转写交互
- 原始音频只保留到转写完成，随后删除

## 10. 全局交互语言

已确认使用 `Ink Appears`。

### 10.1 时间参数

- 直接反馈：120ms
- 内容落定：220ms
- 墨迹显现：320-420ms
- 段落错开：60-80ms

### 10.2 允许的运动

- 2-6px 的纸张或文字位移
- 墨迹显现
- 新线段绘制
- 纸层轻微落定
- 一次性压印

### 10.3 禁止的运动

- 循环脉冲
- 弹跳
- 彩纸或庆祝粒子
- 大幅翻页
- 每次悬停都触发复杂动画
- 将等待状态做成科技感扫描

### 10.4 减少动效

`prefers-reduced-motion` 下：

- 所有状态立即可读
- 保留颜色和层级变化
- 取消位移、线段绘制和错开动画

## 11. 状态与异常

每个页面必须设计：

- 初次加载
- 空状态
- 部分接口失败
- 完全离线
- 提交中
- 提交失败和重试
- 过期数据
- 权限或 provider 不可用

统一原则：

- 已加载内容不因局部失败消失
- 用户输入失败时可以恢复
- 错误写在相关纸页边缘，不使用全屏告警
- 不存在的功能入口不显示
- 被禁用的操作必须说明原因

## 12. 数据与安全逻辑

- 记忆写入必须继续经过蒸馏和优先级规则
- 用户可以纠正画像
- 用户可以知道一条留痕为什么被带回
- 原始置信度、规则码和内部 provider 信息不直接展示
- 整理操作必须先草稿、后确认
- 成就/印记解锁必须幂等并持久化
- 数据迁移前必须自动备份

## 13. 建议删除的旧设计

- 独立行动主导航
- 独立反思主导航
- Steam 式公开稀有度
- 成就完成百分比
- streak 和周/月 KPI
- 首页状态仪表盘、波形和大量状态卡
- 永久 quick prompt 条
- 装饰性设置图标
- 未实现的麦克风按钮
- 空的 `knowledge_base` 能力

## 14. 决策记录与剩余决策

### 已确认

- `D1 = 1`：印记位于留痕内的完整子页，并在首页/学习页显示最近印记
- `D3 = 1`：取消独立行动页，功能分散到首页、学习和整理
- 导航修订：三个长标签为主页面，方形短标签为设置
- 整理入口修订：整理使用上下文入口打开独立覆盖纸，不占标签

### 新增确认

- `R1`：轻读定义为新回复自动朗读总开关；关闭时只输出文字
- `R2 = 1`：语音输入采用云端 STT，转写后写入输入框，不自动发送
- 首个语音供应商使用 SiliconFlow，后续保留其他 provider 适配能力
- `R3 = 1`：轻读首次默认关闭，之后记住用户选择
