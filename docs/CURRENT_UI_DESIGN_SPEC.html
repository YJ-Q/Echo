这份规范以“图像表达优先”为目标，直接把当前系统的所有页面、元素、组件和强弱关系，转换成可视化版面、缩略结构图和部件图库。

Current Direction

温暖纸张、微光在场、对话居中、工作台退后

当前真实系统已经从冷色控制台感，转向“关系空间”方向。规范需要持续保护这几个判断：Arrival 不是任务卡，Hero Orbit 不是状态仪，Conversation Board 不是客服窗，Status Column 不是 dashboard。

主场是对话 状态先被接住 连续性用文字表达 右侧只保留辅助轨

02 Hierarchy

视觉层级热区图

这部分不讲功能先后，而讲用户扫视时的注意力重心。越靠前的层级越应该轻声但明确地占据视线。

左侧导航：常驻，但不要抢戏。它只负责切换，不负责叙事。

中间主区：对话面板与输入区是第一视觉中心；Arrival 只做安静引入。

右侧状态栏：退为边栏注释感，提供“还在场”的线索，而不是数据控制台。

层级 1

最新 Echo 回复、输入框、对话流的呼吸节奏。

层级 2

Arrival 的情绪标题和当下承接说明。

层级 3

右侧专注、下一步、学习线等辅助信息。

层级 4

导航、窗口信息、时间与同步状态等系统边缘信息。

09 Material

材质方向正式决策

当前系统的实物质感正式收敛为“纸面主方向 + 匀光辅方向”。这里不是情绪词，而是后续所有组件调整都要共同服从的材质判断标准。

主方向

温润棉纸、暖白纸面、轻微纤维噪点、半透明描图纸 / vellum 覆层。

这套材质负责“接住、续上、痕迹、可返回”，也是当前 Echo 的全局主表面。

局部吸收

哑光陶瓷 / 磨砂匀光，只用于在场感、hover、focus、typing、Hero Orbit。

这套材质不做主背景，只做“从材质下方透出来的微光”，避免重新变回科技状态仪。

明确排除

石膏 + 原木、厚木板、玻璃、高光塑料、金属控制台、重器物拼装逻辑。

这些方向会把 Echo 拉回 dashboard、工作台、效率工具或器物展台感，不适合作为当前主路线。

10 Input Ritual

对话输入区器物化

这组器物语言只属于对话输入区。它不是新的全局材质系统，而是压在纸面之上的识别锚点：用极少量、极克制的器物感，把“这里可以慢下来开口”变得更可触、更可记住。

发送按钮

圆形、淡绿色、玉质、可轻按，但绝不珠宝化。

主识别点
  • 应更像低饱和冰种软玉，带乳白微浊感和内部匀光，而不是高饱和翡翠珠宝。
  • 边缘应微钝、略圆、极浅厚度，像经打磨后的按压面，不应出现金属包边或玻璃高光。
  • 按下反馈应表现为轻阻尼和极小下沉，不应像机械按钮那样脆弹。

语音按钮

白底、中间一点红、像白玉中嵌着一个收音核。

次识别点
  • 白底应偏羊脂白瓷 / 白玉，中心红点应像朱砂核晕在材质内部，而不是硬边红色图标。
  • 录音中的反馈更适合内部微弱扩散，不适合 LED 闪烁、波形图或工业录音按钮逻辑。
  • 最该避免的是印章化、国旗化、工业麦克风化。

镇纸输入框

整体更方正，只有一点倒角，像压平纸面的轻质案头器物。

稳定器
  • 更适合“压平纸器 + 极轻陶”的混合感，而不是石碑、钢条或大理石条。
  • focus 不应靠粗描边，而应靠镇纸下方纸面被唤醒的柔和匀光。
  • 最该避免重石材纹理、深灰玄武岩感、强 inset shadow 和办公桌工具气质。

还能独特化的点

只挑极小部件做记忆点，不把整片对话区做满。

点睛
输入光标
刻度擦痕
纸面压痕线
  • 输入光标：可做成矿物微晶落点式的柔和呼吸，而不是生硬闪烁黑线。
  • 输入框与历史对话的分界：可做成纸面盲压痕，而不是工业分割线。
  • 状态提示：可做成刻度擦痕或器物表面的轻微痕迹，而不是进度条和数字监控。

必须克制的地方

器物化只在底部输入区成立，不能沿着页面蔓延。

红线
保持干净
不要器物化
不要扩散
  • 主对话历史背景必须继续保持纯净温润纸面，不要加大面积纹理或器物影子。
  • 小工具、快捷按钮、顶栏、导航栏不能全部玉石化，否则会从“识别点”变成“摆件堆”。
  • 器物语言的全屏占比应保持很低，集中在输入区这一处即可。

11 Rules

最该落实到界面的规则

这里保留少量文字，是因为它们将直接影响你后续调界面时的判断。规则配图优先，但每条都必须足够明确，能落到 HTML/CSS/DOM 层。

Rule 01

Arrival 永远小于对话区

页首承接层只能定调，不能重新长成主 hero 卡。视觉体量、底色对比、按钮力度都必须低于 Conversation Board。

Rule 02

禁止进度条化连续性

任何延续、进度、学习推进，都优先用文字状态说明，避免 track、百分比、分段块、完成率环。

Rule 03

Presence 只能是微光

.hero-orbit 只能表达轻微在场，不能有旋转环、雷达圈、虚线仪表、科幻核心感。

Rule 04

对话区不做重盒子

.conversation-board 要尽量贴近页面本体。容器边界能弱就弱，重点靠留白、排版、层级来成立。

Rule 05

消息回合需要停顿感

#timeline-panel 的回合间距要大于传统 IM，Echo 气泡比用户更轻，不要两侧同等硬度。

Rule 06

打字态必须文字化

当前统一使用 .typing-line，它应是低对比、低频、低压力的文本提示，不再使用跳点动画。

Rule 07

右侧栏是批注，不是工作台

.status-column 保留信息，但必须削弱卡片感、指标感、执行台感，像页边注释一样安静退后。

Rule 08

主色是纸与墨，不是蓝色系统

蓝色只能作为低浓度陪衬。主感受必须来自纸白、暖灰、墨色和柔和高光。

Rule 09

按钮优先像邀请，不像命令

Quick prompt、继续动作、侧栏跳转按钮都应该轻。真正强按钮只保留给用户主动提交。
