把 Echo 约束成一个能接住情绪、承接对话、再轻轻推进行动的桌面空间

Echo 不应该长成任务控制台，也不应该像数据仪表盘。当前系统最重要的体验目标，是让用户愿意回来、愿意继续说、愿意顺着眼前这条线再往前一点。

Current Focus

先接住此刻，再慢慢续上

主对话永远是第一焦点，信息结构只为承接对话服务。

01 Principles

原则与定位

先陪伴，再组织

这一部分定义 Echo 的产品气质与 UI 判断标准。只要后续设计方案违背这里的原则，即使视觉更炫，也不应进入主产品。

Product Position

Echo 是陪伴空间，不是任务系统

界面需要同时承接四件事：接住用户状态、保持主对话、保留当前主线、用轻量结构支持学习与成长记录。

  • 判断标准不是“信息更多”，而是“用户更容易回来、更容易继续说、更容易接上当前这条线”。
  • 学习、行动、反思、记忆都属于主对话的延伸层，不得夺走系统中心。
  • 推荐、提醒、任务安排都要像陪伴式提议，而不是控制式派发。

Experience Rules

当前版本的四条体验原则

  • 先接住，再引导。
  • 先对话，再管理。
  • 先保留主线，再展开信息。
  • 先轻量可继续，再补足层级与结构。

Should Feel Like

应当呈现的气质

  • 安静，不吵闹。
  • 温和，不命令。
  • 清晰，但不过度硬朗。
  • 轻微活着，有呼吸感，但不过度拟人表演。

Must Avoid

明确要排除的表达

  • 仪表盘式任务管理感。
  • 炫技 AI 面板感。
  • 情绪表演过重的“陪聊宠物”感。
  • 多个同等级焦点同时争夺注意力。

03 Layout

布局骨架

三层桌面结构

当前桌面端使用稳定的三层骨架。这个结构不是为了“信息最大化”，而是为了让产品存在感、对话空间与连续性提示各司其职。

Shell Structure

主骨架职责

  • nav-rail：页面切换与产品存在感，宽度固定，视觉上更像呼吸中的书签，而不是功能树。
  • main-window：承载所有核心信息，是用户主要停留区域。
  • status-column：只在 Now 页作为轻量侧轨出现，用来保留连续性，不承接主任务流。

Window Logic

主窗口结构

  • app-titlebar：桌面身份与同步状态。
  • topbar：当前视图名称、说明与时钟信息。
  • workspace-view：页面实际内容区，各视图布局遵守同一节奏。

04 Components

核心组件规则

对话必须是系统主角

Conversation Board

主对话板

对话区必须始终像一个可停留、可继续、可回来的空间，而不是即时消息列表或客服工单面板。

先接住你现在的状态，我们沿着刚才那条线慢慢续上。
我有点卡住了，但还是想继续往前推一点。
那就不扩太大，先把下一步缩成一个现在就能做的动作。
Echo 正在回应...
  • Echo 气泡左侧落位，用户气泡右侧落位，强化“陪伴式并行”而非对抗问答。
  • 消息宽度不满屏，保留呼吸边界。
  • 打字态必须轻，不做夸张动画和高频闪烁。

Composer

输入区与快捷提词

输入区是延续对话的入口，不是命令行。快捷提词要像“给用户搭把手”，而不是催促按钮。

已完成上一步 我卡住了 继续当前线索
继续输入你现在最想先理清的一件事...
朗读最新回复 发送
  • 快捷提词不超过 3 个，且必须贴近当前情境。
  • 主按钮清晰可见，但不应做出强推进感的高饱和强调。
  • 任何“推荐任务”都优先通过对话语境进入，而不是直接替换输入区中心地位。

Arrival Cards

Now 页到达卡片

  • 到达卡负责承接“你现在在哪里”，不是展示所有统计。
  • 左侧为情绪与状态陈述，右侧为当前焦点与下一步。
  • 卡片中的 CTA 只允许一条主延续动作，例如“把这一步带回对话”。

Status Column

右侧侧轨

  • 右侧只保留“还活着的线”，不堆叠冗余模块。
  • 用于轻提醒、连续性提示、成就回声，不用于主操作。
  • 在较窄屏幕下可以隐藏，证明它不是主体验依赖项。

05 Pages

页面规则

一个页面只保留一个最强焦点

Now

此刻页是系统中心

Now 页负责承接状态、继续主对话、保留当前主线，是整个产品最重要的一页。

状态到达 主对话流 轻量连续性侧轨
  • 主对话必须处于页面视觉中心，并且留出足够纵深。
  • 任何学习、任务、记忆信息进入 Now 页时都只能以“下一条线索”形式出现。

Learn / Actions / Reflections / Memory

四个专题页是延伸层

这些页面允许结构化信息更完整，但仍然必须保持“当前一步”逻辑，不能长成独立的管理后台。

当前学习线 当前任务队列 近期反思 重要记忆层
  • 每页只强调一个当前动作或当前主题。
  • 允许从专题页把内容“带回对话”，不鼓励在专题页内闭环一切。

06 Motion

动效与响应式

轻微活着，不喧宾夺主

Motion Rules

动效规范

Fast / 120ms 按钮 hover、chip 切换、轻微状态反馈。
Base / 180ms 卡片层级变化、导航切换、输入反馈。
Slow / 260ms 页面级淡入、启动页消退、情绪光晕等呼吸感动画。
  • 曲线统一采用柔和减速，不使用弹跳或激进位移。
  • 动画的目的应是“让状态被感觉到”，而不是“展示产品很聪明”。

Responsive Rules

响应式优先级

  • 先保留主对话与主内容，再考虑压缩装饰区。
  • 窄屏下右侧侧轨可隐藏，证明其始终是从属结构。
  • 多栏页面收束为单栏时，保持内容顺序不打乱“当前一步”的逻辑。
  • 移动端如果后续单做版本，应重新定义骨架，而不是机械缩放桌面布局。

07 Checklist

验收清单

讨论新方案前先过这一轮

Function + Experience

功能落地时要检查

  • 是否仍然以主对话为中心，而不是让新功能夺走主舞台。
  • 是否帮助用户更容易回来、继续说、继续做，而不是增加额外理解成本。
  • 是否把推荐、提醒、任务安排表达成提议，而不是指令。
  • 是否保留了“当前主线”这一唯一最强焦点。

Visual + Tone

界面更新时要检查

  • 有没有产生仪表盘感、管理后台感、AI 炫技面板感。
  • 颜色、动效、文案有没有把产品变得太兴奋、太命令、太表演。
  • 页面焦点是否过多，是否存在多个区域同时大声说话。
  • 新增样式是否仍与当前白雾蓝调、圆角玻璃层级保持一致。
