# Echo 产品地图

本文档用于梳理 Echo 当前版本的整体交互逻辑、页面职责、功能边界和后续推进顺序，作为前后端和设计继续协作的统一基准。

## 1. 产品定位

Echo 不是一个传统聊天工具，也不是一个重型任务管理器。

它更像一个长期陪伴型桌面体，核心价值有三层：

1. 接住用户当下的状态。
2. 把模糊的压力、任务和学习线索收束成下一步可执行动作。
3. 通过记忆和反思，逐渐形成连续性的“第二自我”。

因此，整个产品要始终围绕三个关键词展开：

- 陪伴感
- 连续性
- 推进感

## 2. 当前版本的主结构

当前桌面端以五个页面构成：

1. `此刻`
2. `学习`
3. `行动`
4. `反思`
5. `记忆`

其中：

- `此刻` 是主驾驶舱，承担实时状态、对话和下一步推进。
- 其余四页是围绕主流程展开的专题工作页。

## 3. 全局交互逻辑

### 3.1 主入口

用户进入产品后，默认进入 `此刻` 页。

这一页需要同时完成四件事：

1. 告诉用户当前 Echo 理解到的状态。
2. 展示当前最值得推进的一条主线。
3. 提供继续对话的最低摩擦入口。
4. 让用户在不切页的前提下，随时回到“下一步动作”。

### 3.2 产品主循环

当前版本的产品循环可以概括为：

1. 用户输入一句话或使用快捷提示。
2. 系统分析情绪、意图、学习线索和任务线索。
3. 系统返回 Echo 回复，并同步更新当前状态。
4. 后端把信息沉淀到任务、学习、记忆、反思这四条结构化链路。
5. 前端重新拉取状态，把变化反映到五个页面。

也就是说，产品真正的主引擎不是某个单页，而是：

`对话 -> 理解 -> 结构化沉淀 -> 再反馈到界面`

## 4. 五个页面的职责边界

### 4.1 此刻

目标：

- 承接当下
- 继续对话
- 明确下一步

包含的模块：

1. 当前状态卡
2. 下一步行动卡
3. 主对话时间线
4. 快捷提示和输入框
5. 右侧状态工作台

主要动作：

- 发送消息
- 用快捷提示继续当前线索
- 把当前行动带回对话

边界：

- 这里不做复杂历史筛选。
- 这里不做重型任务编辑。
- 这里不做深层记忆管理。

### 4.2 学习

目标：

- 展示当前学习主题
- 呈现步骤序列
- 推进当前一步

包含的模块：

1. 学习主题头部
2. 步骤列表
3. 当前一步说明
4. 当前一步操作按钮

主要动作：

- 把某一步设为进行中
- 把某一步设为完成

边界：

- 这里不承担自由聊天。
- 这里不需要复杂课程体系。
- 这里先聚焦“当前一条学习线”，而不是多条并行课程。

### 4.3 行动

目标：

- 明确当前主任务
- 展示剩余任务队列
- 支持手动补任务或让系统生成建议任务

包含的模块：

1. 当前主任务卡
2. 建议任务入口
3. 手动任务入口
4. 队列列表

主要动作：

- 设为进行中
- 标记完成
- 移除任务
- 创建手动任务
- 生成建议任务

边界：

- 当前版本不做拖拽排序。
- 当前版本不做复杂优先级编辑面板。
- 当前版本优先保证“主任务 + 队列 + 状态切换”这条链闭环。

### 4.4 反思

目标：

- 呈现最近的总结
- 展示情绪趋势和行为模式
- 提供手动刷新总结入口

包含的模块：

1. 最新总结头部
2. Echo 反思摘要
3. 行为模式摘要
4. 历史反思卡片

主要动作：

- 刷新总结

边界：

- 当前版本不做复杂可视化分析大屏。
- 当前版本不做时间跨度筛选。
- 当前版本先把“最近几条反思”读清楚即可。

### 4.5 记忆

目标：

- 让用户看到当前被系统保留的关键线索
- 让用户对重要记忆做轻量干预

包含的模块：

1. 记忆概览
2. 标签热度
3. 记忆分层统计
4. 核心 / 重要 / 环境记忆列表

主要动作：

- 置顶记忆
- 提升记忆优先级

边界：

- 当前版本不做全文搜索。
- 当前版本不做复杂标签体系管理。
- 当前版本不做多层级知识图谱。

## 5. 当前产品的四条核心数据链

### 5.1 对话链

作用：

- 承接用户当下输入
- 给出 Echo 回复
- 驱动状态刷新

前端表现：

- 此刻页时间线
- 快捷提示
- 输入框
- 最新回复朗读

### 5.2 学习链

作用：

- 把“我想学 / 我正在学”的内容转成可推进步骤

前端表现：

- 学习主题
- 步骤序列
- 当前一步动作

### 5.3 行动链

作用：

- 把模糊任务变成当前主任务和队列

前端表现：

- 主任务卡
- 队列列表
- 建议任务 / 手动任务入口

### 5.4 记忆与反思链

作用：

- 让系统形成连续性
- 让用户能看到被保留了什么、最近重复发生了什么

前端表现：

- 记忆页
- 反思页
- 此刻页右侧趋势摘要

## 6. 当前版本的状态流转

### 6.1 用户状态流转

当前用户状态可以粗分为：

- `anxious`
- `motivated`
- `focused`
- `distracted`
- `neutral`

这些状态目前影响三个地方：

1. `此刻` 页的当前状态展示
2. 右侧状态工作台
3. Echo 的回复语气

### 6.2 任务状态流转

任务状态：

- `pending`
- `active`
- `done`
- `dismissed`

前端上最关键的流转逻辑是：

1. 队列里可以有很多任务。
2. 但同一时刻应该只强调一个主任务。
3. 任务状态改变后，界面要重新计算主任务和剩余队列。

### 6.3 学习步骤流转

步骤状态：

- `pending`
- `active`
- `done`

前端逻辑：

1. 当前步骤优先高亮。
2. 已完成步骤弱化但保留。
3. 下一步始终要有清楚的动作说明。

## 7. 交互优先级

当前阶段应当优先打磨这些交互：

### P0

- 对话发送和刷新状态
- 主任务展示与状态切换
- 学习步骤状态切换
- 反思刷新
- 记忆置顶 / 提权

### P1

- 更好的空状态提示
- 更好的异常提示
- TTS 使用反馈
- 页面之间的上下文联动

### P2

- 更完整的历史浏览
- 更细的图形反馈
- 更个性化的成长与画像视图

## 8. 当前明显缺口

从产品层面看，当前还存在这些明显缺口：

1. `此刻` 页的状态呈现还偏静态，缺少更强的“活着的状态信号”。
2. `学习` 页和 `行动` 页已经能操作，但还不够强调“当前只做哪一步”。
3. `反思` 页和 `记忆` 页能读数据，但还需要更强的信息层级。
4. 整个产品缺少一条更明确的“成长反馈线”，也就是用户长期为什么愿意回来。

## 9. 下一阶段建议

建议按下面顺序继续推进：

1. 先把 `此刻` 页打磨成稳定主驾驶舱。
2. 再强化 `学习` / `行动` 的当前一步表达。
3. 然后补 `反思` / `记忆` 的图形化层级。
4. 最后再考虑“成长页 / 成就感 / 长期画像”这类二阶段内容。

## 10. 本轮前端工作的判断标准

当我们继续补前端时，可以用下面四条判断“是不是对的”：

1. 用户是否能在 3 秒内看懂当前最重要的东西。
2. 用户是否能在 1 次点击内回到“下一步”。
3. 界面是否始终围绕一个主线推进，而不是并列很多信息。
4. Echo 是否看起来像一个持续理解我的陪伴体，而不是一堆模块的拼装界面。
