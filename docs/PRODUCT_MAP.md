# Echo 产品页面与模块地图

本文件用于统一 Echo 桌面端的页面目标、模块边界、交互优先级，并作为前后端协同开发的基准。

## 1. 全局原则

- 产品保持简单、低负担、强推进感
- 产品必须突出长期陪伴与情绪价值
- 主流程优先于辅助信息
- 一页一个目标
- 一屏一个主动作
- 后端能力优先服务明确页面模块，不为抽象能力而开发

补充说明：

- Echo 首先是陪伴式交流体，其次才是任务推进器
- 任务、记忆、反思都服务于“逐渐成为另一个我”这一长期目标

## 2. 页面清单

### 2.1 此刻

页面目标：

- 让用户看清当前状态
- 和 Echo 继续对话
- 承接当前主任务或下一步行动
- 在交流中处理困惑、获得理解与陪伴

页面定位：

- Echo 的主驾驶舱
- 唯一保留三栏布局的页面

核心模块：

1. 当前状态主视觉
2. 当前主任务摘要带
3. 聊天框式主对话流
4. 多模态输入区
4. 右侧状态工作台

主动作：

- 发送消息 / 继续推进

此页补充要求：

- 中间主区必须像舒服的聊天场域，而不是任务日志
- 时间戳放在消息角落弱化显示
- 快捷提示要偏“陪伴推进型”，而不是偏命令式

### 2.2 学习

页面目标：

- 展示当前学习主题
- 展示步骤推进情况
- 推进当前学习步骤

页面定位：

- 学习任务线页面

核心模块：

1. 学习主题头部
2. 步骤序列
3. 当前步骤详情
4. 学习模式 / 行为提示

主动作：

- 推进当前步骤

### 2.3 行动

页面目标：

- 让用户明确当前主任务
- 展示次级任务队列
- 开始或完成当前任务

页面定位：

- 任务执行页面

核心模块：

1. 当前主任务卡
2. 任务队列
3. 阻碍与辅助
4. 任务进度表现

主动作：

- 开始 / 完成当前任务

### 2.4 反思

页面目标：

- 展示近期总结
- 呈现趋势与主导模式
- 刷新当下反思结果

页面定位：

- 总结与回顾页面

核心模块：

1. 近期总结
2. 趋势图
3. 主导模式卡
4. 历史摘要

主动作：

- 刷新总结

### 2.5 记忆

页面目标：

- 展示关键召回片段
- 呈现记忆分组与标签热度
- 允许置顶或提升重要记忆

页面定位：

- 召回与沉淀页面

核心模块：

1. 记忆概览
2. 分组 / 聚类
3. 关键片段列表
4. 标签热度

主动作：

- 查看 / 置顶关键记忆

## 3. 状态工作台模块

右侧状态工作台仅属于 `此刻` 页。

固定保留模块：

1. 当前状态
2. 专注
3. 下一步行动
4. 学习线
5. 成长进度
6. 记忆概览或反思趋势（二选一弱化）

模块要求：

- 每张卡只表达一种信息
- 每张卡最多一个主动作
- 不重复中间主区已表达的内容

### 3.1 成长进度卡

页面定位：

- 陪伴式成长证据
- 长期回访价值入口

建议内容：

- 本周完成量
- 本月完成量
- 连续推进热力格或格点
- 一句轻里程碑总结

不建议：

- 做成重统计报表
- 做成纯数字 KPI 卡

## 4. 模块级数据需求

### 4.1 当前状态主视觉

需要数据：

- 当前状态名
- 情绪类型
- 简短说明
- 当前模式 / 注意事项

前端表现：

- 状态文字
- 心跳波形
- 一句简短说明

### 4.2 主对话流

需要数据：

- 用户消息
- Echo 回复
- 时间
- 标签 / 轻状态
- 可选语音相关状态

前端表现：

- 聊天框式对话流
- 输入中的过渡状态
- 新回复出现反馈

补充要求：

- Echo 回复要更像陪伴式交流，不像工单回复
- 对话流需要能够承接任务生成、困惑澄清、生活与学习对话

### 4.2.1 多模态输入区

需要数据 / 能力：

- 文本输入
- 语音输入状态
- 语音输出状态
- 发送中状态

前端表现：

- 文本输入框为主
- 语音入口明确但克制
- 不使用复杂音频控制台

### 4.3 当前主任务卡

需要数据：

- 任务标题
- 任务说明
- 当前步骤
- 完成条件
- 进度值
- 状态

前端表现：

- 明确像 quest 的任务卡
- 标题清晰
- 进度直观
- 只有一个主按钮

补充要求：

- 文案必须像同行提示，而不是命令
- 任务卡应从对话关系里自然长出来

### 4.4 学习步骤序列

需要数据：

- 学习主题
- 总步骤数
- 当前步骤索引
- 每一步标题
- 每一步状态

前端表现：

- 垂直步骤流或分段进度
- 已完成 / 当前 / 未完成状态明确

### 4.5 反思趋势

需要数据：

- 最近总结
- 情绪趋势
- 行为模式
- Echo 的反思文字

前端表现：

- 小趋势线
- 简短总结卡
- 模式标签

### 4.6 记忆模块

需要数据：

- 记忆条目
- 标签
- 重要性
- 置顶状态
- 最近访问时间

前端表现：

- 网格 / 聚类
- 简化列表
- 少量操作

## 5. 核心交互清单

### 5.1 此刻页

必须有：

- 发送消息
- 点击快捷提示填充输入
- 点击下一步行动跳转或填入输入
- 查看当前状态

可后置：

- 播放 TTS
- 对话中的更多历史筛选

### 5.2 学习页

必须有：

- 查看当前主题
- 查看步骤
- 将步骤标记为进行中 / 完成

可后置：

- 学习路径详情展开
- 学习成就记录

### 5.3 行动页

必须有：

- 查看任务队列
- 将任务设为进行中
- 将任务标记完成

可后置：

- 手动重排任务
- 更复杂的任务过滤

### 5.4 反思页

必须有：

- 查看近期总结
- 刷新总结

可后置：

- 历史反思筛选
- 更多模式对比

### 5.5 记忆页

必须有：

- 查看记忆列表
- 置顶记忆
- 提升记忆优先级

可后置：

- 复杂的标签管理
- 更多召回方式

## 6. 轻游戏化机制地图

Echo 可以借用游戏设计理念，但必须克制。

首版推荐保留：

1. 主任务 / 当前步骤
2. 明确完成条件
3. 直观进度反馈
4. 大任务完成后的轻反馈卡
5. 一周 / 一月成长进度表现

二期可加入：

1. 成就系统
2. 成长页 / 个人档案页
3. AI 生成的成就描述
4. 任务结算卡
5. 成长里程碑展示区

三期再考虑：

1. 连续性奖励
2. 稀有成就
3. 解锁式成长内容

## 7. 文档更新规则

本文件是“页面与模块基准”。

后续如有以下变化，必须同步更新：

- 页面目标变化
- 模块增删
- 交互优先级变化
- 首版 / 二期边界变化

不要默认整仓覆盖代码。

推荐流程：

1. 先同步最新远端进度
2. 对照 `PRODUCT_MAP.md`
3. 明确本次负责模块
4. 再修改代码
