# Margin 功能审计 V1

日期：2026-07-10

状态：审计结论已纳入批准的产品蓝图

## 1. 审计范围

本次审计交叉检查了：

- 当前 React 前端
- Express 路由和 service 层
- SQLite 数据表
- 自动化测试
- Echo 时代的信息架构、前端映射和执行文档
- Git 历史中的旧 `public/` 前端
- 当前本地数据库的只读状态

处理结论分为五类：

- `保留`：产品方向和实现均合理
- `重构`：能力值得保留，但入口、文案或交互不适合 Margin
- `合并`：不再作为独立页面，合入更合适的流程
- `延期`：有价值，但当前缺少前置能力或隐私定义
- `删除`：旧设计遗留、无实际实现或与当前定位冲突

## 2. 结论摘要

当前项目不是缺少后端能力，而是 React 迁移后把多个完整功能压缩进了一个通用列表组件。

最明显的遗漏包括：

- 成就墙、隐藏成就、最近解锁提示和图标资产
- 学习事件历史
- 当前行动、建议行动和行动创建
- 记忆置顶、重要性调整和长期画像纠正
- 整理草稿取消、操作历史和可恢复流程
- 用户可读的决策原因

同时存在若干旧功能或占位功能，需要删除或延期：

- 独立任务管理页
- 独立反思页
- 周/月完成量、连续天数和 streak 式成长统计
- 无任何调用方的 `knowledge_base` 表
- 未实现的麦克风入口
- 无交互的设置图标
- 被误用为整理入口的 `+` 标签

## 3. 成就功能专项审计

### 3.1 已经存在的部分

后端已存在：

- `GET /achievements`
- `GET /achievements/recent`
- `GET /achievements/icons`
- 8 个固定成就定义
- 学习、行动、记忆和整理事件推导出的解锁状态
- 隐藏成就字段、来源字段、稀有度字段和图标类型字段
- 3 组 API 自动化测试

当前本地数据可以推导出至少 3 个已解锁成就：

- 创建第一条学习线
- 完成学习线第一步
- 留下长期记忆锚点

### 3.2 被 React 迁移遗漏的部分

当前前端只做了以下处理：

- 请求完整成就列表
- 只筛选已解锁项
- 最多取 3 项
- 把它们作为普通文本列表插在留痕页中

因此用户看不到：

- 完整成就集合
- 未解锁线索
- 隐藏成就揭示
- 最近解锁的独立反馈
- 学习线附近的相关成就
- 成就图标
- 成就来源和解锁日期的完整表达

### 3.3 还没有真正完成的部分

- 没有 `achievements` 和 `achievement_unlocks` 持久化表
- 解锁状态每次由当前数据重新推导，不是不可丢失的历史事实
- 没有按具体学习主题生成成就
- 没有事件级幂等解锁记录
- 图标目录引用的 7 张图片实际不存在
- 没有专用前端页面或完整子页面

结论：成就功能是“后端基础已实现，产品功能只完成约一半，前端表达被遗漏”。

### 3.4 推荐重构

用户可见名称建议从“成就”改为 `印记`。

保留：

- 关键行为被看见
- 隐藏印记
- 主题相关印记
- 最近获得的轻微惊喜
- 完整收藏视图

删除或隐藏：

- 分数
- 完成百分比
- 排名
- streak
- 公开的 common / rare / core 等稀有度标签

稀有度可以继续作为内部视觉权重，但不直接展示给用户。

推荐入口：

- 首页只出现一次最近获得的印记
- 学习页显示与当前学习线相关的 1-2 枚印记
- `留痕` 内提供完整 `印记` 子页

印记视觉建议采用压印、墨章或纸上凹凸印，而不是游戏徽章卡片。

## 5. 文档和契约漂移

需要在方案确认后统一修正：

- 当前 API 文档把已经存在的成就接口写成“未来建议”
- API 文档存在 `GcT`、`aemory` 等损坏标题
- 备份文档没有列出实际已导出的 `operation_proposals` 和 `operation_events`
- README 声称 achievement 已完整连接，但当前只显示普通文本条目
- 图标目录返回不存在的资产路径

## 6. 不应直接删除的数据能力

以下功能即使不做主页面，也应保留后端能力：

- actions
- user states
- profile
- explainability
- operation audit events
- achievement source fields

它们会继续支持对话、学习、留痕和整理，不等于必须拥有独立导航页。

## 7. 方案确认前冻结项

在产品蓝图确认前，不进行：

- 新一轮 image generation
- 页面组件重写
- 数据表删除或迁移
- 成就持久化实现
- 导航结构变更
- 整理执行器扩展
