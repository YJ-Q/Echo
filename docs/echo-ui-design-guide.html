Echo UI Design Guide

深空中的回声生命，以星体、轨道和粗糙信号片段构成界面。

这份指南把当前方向拆成可复用的 UI 元素：深空背景、星图侧边栏、白色 Echo 星体、粗糙气泡、星球表面输入台。后续 HTML 原型应从这些元素组合，而不是回到通用聊天界面。

深空为场背景是沉默空间，不是装饰星空。光点要少、慢、远。
Echo 是星体白色圆形、柔和光晕、轻微回声环，不拟人。
消息是信号片气泡像粗糙星岩碎片，灰白与淡蓝区分角色。
导航是星图侧边栏采用轨道节点，不使用说明文字和普通图标。

色彩令牌

整体使用深黑宇宙、灰白星岩、淡蓝用户信号和白色 Echo 光源。避免绿色、橙色、紫色和高饱和赛博霓虹。

侧边栏规则

采用用户参考图的结构：当前节点发光、节点之间虚线连接、图标为手绘星体符号。默认只保留图标，不显示文字。

  • 默认状态只显示图标，不显示“此刻/学习/行动/反思/记忆”。
  • 文字只允许作为 hover tooltip 或无障碍标签存在。
  • 当前项使用白蓝发光圆形，其他图标使用粗糙白线。
  • 轨道线必须细、断续、低对比，不抢 Echo 本体。

消息气泡

气泡是漂浮在深空里的“信号片”，不是标准 IM 气泡。边缘要粗糙、有颗粒、轻微不规则。

用户消息：淡蓝色，代表向深空发出的信号。
Echo 回声：灰白色，像从星体表面返回的片段。

输入信号台

输入框是白色粗糙方框，像星球表面的信号台。它是用户向 Echo 发出信号的地方。
