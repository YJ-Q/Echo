# Margin Memory Layers

Date: 2026-07-07

## Goal

Define what Margin should remember, what it should not overremember, and how memory should support companionship and growth without feeling invasive.

Margin should not treat every message as a permanent memory.

The memory system should preserve:
- continuity
- emotional safety
- growth visibility
- relevance

## Core Principle

Not every message deserves the same memory weight.

Margin should use layered memory:

1. short-term conversation context
2. mid-term growth fragments
3. long-term core memory

## Layer 1. Short-Term Context

Purpose:
- maintain conversational continuity
- keep the current thread coherent

What belongs here:
- the recent exchange
- current emotional tone
- the current topic
- the active learning or action line

Examples:
- what the user is talking about right now
- whether the user is venting or asking for help
- the current learning step

Retention style:
- temporary
- recent-window based
- easy to replace

Should not automatically become long-term memory.

## Layer 2. Growth Fragments

Purpose:
- capture meaningful progress patterns across time

What belongs here:
- repeated friction points
- repeated recovery patterns
- meaningful insights the user expresses
- completed growth moments
- learning milestones

Examples:
- often gets stuck at the start
- responds well to very small next steps
- moved from overwhelm to naming the real blockage
- completed the first working example in a topic

Retention style:
- promoted only when repeated or meaningful
- summarized rather than stored verbatim

This is the most important layer for Margin's product identity.

## Layer 3. Core Memory

Purpose:
- form the long-term second-self understanding

What belongs here:
- durable learning interests
- stable motivational patterns
- important self-definitions
- repeated emotional signatures
- major breakthroughs

Examples:
- current long-term learning line
- common start friction
- tends to recover better when the next step is tiny
- important recurring values or ambitions

Retention style:
- curated
- sparse
- high-confidence only

Core memory should feel earned, not automatic.

## What Should Usually Not Be Strongly Remembered

By default, Margin should avoid heavily storing:

- one-off mood blurts
- casual small talk
- low-signal filler
- temporary irritation without recurrence
- every single vulnerable sentence as a permanent identity fact

Why:
- the user should feel accompanied, not surveilled
- temporary states should not become permanent labels

## Promotion Rules

Messages or patterns can move upward only when they earn it.

Suggested promotion logic:

### From short-term context to growth fragment

Promote when:
- the same issue repeats
- the user explicitly names a meaningful insight
- a real learning or action milestone happens
- the same blockage appears across sessions

### From growth fragment to core memory

Promote when:
- the pattern is stable
- the evidence repeats across time
- it meaningfully affects how Margin should respond in the future
- it helps the product feel personally adaptive

## Memory By Dialogue Mode

### Vent

Default behavior:
- store lightly
- prefer temporary emotional context

Promote only when:
- the same emotional pattern repeats
- the user expresses an important self-insight
- the emotional moment is clearly tied to a growth pattern

### Casual

Default behavior:
- store very lightly

Promote only when:
- it reveals a stable preference
- it reveals a meaningful relationship or life pattern
- it becomes part of continuity that actually matters

### Reflect

Default behavior:
- medium memory priority

Often worth capturing:
- self-observed patterns
- clarified causes
- named triggers

### Action

Default behavior:
- high memory relevance

Often worth capturing:
- current next step
- completed step
- stuck point
- recovery move
- learning milestone

## Memory Content Shape

Margin should prefer summarized memory forms over raw quotes.

Preferred forms:
- short memory note
- short insight note
- growth pattern label
- confidence-backed profile signal

This helps memory stay:
- readable
- useful
- less invasive

## Safety And Trust Principle

The user should be able to feel:

- I can talk freely without everything becoming permanent
- Margin remembers what matters, not every passing feeling
- Margin understands patterns, not just snapshots

## Product Implications

This layered design supports:
- stronger companionship
- more accurate growth summaries
- more trustworthy long-term personalization
- less memory pollution

It also reduces two major risks:

1. overremembering casual or vulnerable content
2. confusing current emotion with stable identity

## Success Standard

The memory system is working when:

- Margin feels continuous
- Margin feels respectful
- important growth becomes visible over time
- users do not feel excessively recorded
