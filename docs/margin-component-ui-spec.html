Margin 组件化 UI 设计规范

把完整页面拆成 8 个可独立生成、独立验收、最后组装的组件。目标是降低复刻难度，让每个局部都先做准。

复刻原则

先把每个组件单独做准，再组装完整页面。任何生成工具都必须保留：左侧 2/3 对话、右侧 1/3 硫酸纸、4 个标签对应 4 张等尺寸纸页、用户右对齐、Margin 左对齐。

Component Library

1. PaperStage

背景、4 张等尺寸纸页、左侧标签、66/34 版式容器。

  • 纸页数量必须等于标签数量。
  • 每个标签贴在不同纸页边缘。
  • 每层只偏移 2-3px，禁止随机纸堆。

2. DraftHeader

品牌、朗读偏好、设置入口。

  • 轻读是开关，不是播放器。
  • 设置图标在轻读右侧。
  • 禁止菜单化或工具栏化。

3. DialogueFlow

无气泡的纸面对话流。

  • Margin 左对齐，用户右对齐。

4. MessageHoverTools

历史用户消息的上下文编辑工具。

  • 仅 hover 时出现。
  • 橡皮擦删除，铅笔修改。
  • 禁止放入输入框。

5. Composer

默认输入状态，一条纸面落笔线。

  • 只保留语音和发送。
  • placeholder 固定为“写下此刻的想法...”。
  • 禁止边框输入框。

  • 禁止一次展开成大文本框。

Assembly Order

1. PaperStage
先固定纸页层叠、标签和 66/34 空容器。
2. Header + Composer
确认顶部控制和底部输入的纸面语气。
3. DialogueFlow
单独调准用户右、Margin 左的文字柱。
4. Vellum + LearningLine
最后放入硫酸纸、书签和紧凑学习线。
