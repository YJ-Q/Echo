# Margin Documentation

This directory separates current Margin guidance from archived Echo-era design exploration.

## Current Product Documents

- `PRODUCT_POSITIONING_V2.md` - product role and boundaries
- `MARGIN_DESIGN_LANGUAGE.md` - paper, ink, trace, and continuation language
- `VOICE_AND_GUARDRAILS.md` - writing tone and safety boundaries
- `IMPLEMENTATION_ROADMAP.md` - current implementation status and next work
- `PAGE_ARCHITECTURE_V2.md` - page responsibilities, material layers, layouts, and interactions
- `INTERACTION_DIRECTIONS.md` - selected motion language and rejected alternatives
- `FEATURE_AUDIT_V1.md` - implementation, omission, obsolescence, and retention audit
- `PRODUCT_DESIGN_BLUEPRINT_V1.md` - approved source of truth for product, page, flow, material, and interaction decisions

## Current Engineering Documents

- `API_CONTRACT.md` - supported API contract
- `FUNCTIONAL_ACCEPTANCE.md` - verified behavior and resolved findings
- `MEMORY_LAYERS.md` - memory model
- `BACKUP_AND_EXPORT.md` - local data backup and restore
- `RELEASE_CHECKLIST.md` - release checks

## Archive

`archive/echo-era/` contains previous UI concepts, mock-driven frontend plans, Gemini handoff files, and design language that no longer governs Margin. It is retained for history only and must not be used as the source of truth for new UI work.
