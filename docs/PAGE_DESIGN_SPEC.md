# Echo Now Page Design Spec

本文件是 Echo `此刻页` 的模块级设计规范。  
它不是整站概述，而是专门服务于：

- 产品设计出图
- 前端页面实现
- 组件拆分
- 状态映射
- 文案和交互收敛

本文档默认继承 [STYLE.md](/D:/Echo/docs/STYLE.md) 中已经确定的视觉基线，  
当前主方向为 `Breathing Minimal / 留白呼吸`。

---

## 1. 页面定义

### 1.1 页面角色

`此刻页` 是 Echo 的默认入口，也是全产品的主驾驶舱。

它承担 4 个核心职责：

1. 接住用户当下状态
2. 让用户与 Echo 持续对话
3. 把模糊压力收束成一个下一步
4. 把结构化状态映射成可见界面

### 1.2 页面目标

用户打开页面后，应在 3 秒内明确感知到：

- 我现在处于什么状态
- Echo 还在持续跟着我
- 我可以继续说
- 我有一个明确但不压迫的下一步

### 1.3 页面不做什么

此刻页不承担：

- 大量历史会话回看
- 复杂任务编辑
- 学习步骤详细管理
- 反思全文阅读
- 记忆检索系统

这些内容只能在专题页深度操作。  
此刻页只负责主线承接与轻推进。

---

## 2. 页面结构总览

此刻页采用三栏布局：

1. 左侧导航栏 `Navigation Rail`
2. 中部主区 `Conversation Workspace`
3. 右侧状态栏 `Status Workbench`

推荐结构：

```text
App Shell
├─ Navigation Rail
├─ Main Window
│  ├─ Titlebar
│  ├─ Page Header
│  ├─ Hero Area
│  ├─ Conversation Stream
│  └─ Composer
└─ Status Workbench
   ├─ Current State Card
   ├─ Focus Card
   ├─ Next Action Card
   ├─ Learning Card
   └─ Reflection or Memory Card
```

---

## 3. 信息优先级

此刻页的视觉优先级必须固定：

1. 页面主标题与当前语境
2. 对话流
3. 下一步行动
4. 当前状态
5. 学习与其他辅助信息

这意味着：

- 中区对话绝不能被右栏压过
- 右栏是辅助工作台，不是后台
- Hero 区用于定调，不用于塞满数据

---

## 4. 模块拆解

## 4.1 左侧导航栏

### 模块目标

提供稳定、低认知成本的全局切换能力。

### 模块组成

- Echo 标识
- Presence 状态
- 页面切换项

### 必备导航项

- 此刻
- 学习
- 行动
- 反思
- 记忆

### 交互要求

- 当前页面必须清晰高亮
- hover 反馈轻
- 不出现复杂文字解释
- 不在导航栏堆状态数字

### 视觉要求

- 作为全页面最深的容器存在
- 颜色比中区更沉稳
- 激活项用蓝色强调
- 保持窄而稳定

### 尺寸建议

- 宽度：`72px - 88px`
- 内边距：`12px - 18px`
- 单项高度：`48px - 58px`

---

## 4.2 窗口标题栏

### 模块目标

提供桌面应用属性和系统感，不参与核心信息竞争。

### 模块组成

- 产品标识或轻品牌标签
- 当前运行状态
- 同步状态
- 窗口控制按钮

### 信息要求

允许展示：

- `ECHO`
- `本地模式 / 已同步 / 同步中`
- 桌面陪伴体等轻描述

不允许展示：

- 冗长产品介绍
- 复杂统计信息
- 大量系统说明

### 视觉要求

- 低存在感
- 对齐严谨
- 不抢 Hero 和对话的注意力

---

## 4.3 页面头部 Header

### 模块目标

告诉用户当前所在页面和当前语境。

### 模块组成

- 面包屑式标签
- 页面标题
- 一句副标题
- 右侧时间与轻状态

### 内容规则

主标题固定为：

- `此刻`

副标题需要表达：

- 当前是承接当下状态的页面
- 用户可以继续对话
- 页面不会给用户过重压力

推荐语气：

- 承接当前状态、对话和下一步行动
- 先接住此刻，再决定往哪走

### 视觉要求

- 标题区必须有留白
- 不用多色标签抢焦点
- 时间为弱信息

---

## 4.4 Hero Area

### 模块目标

Hero 是此刻页的定调区。  
它不是营销 hero，也不是仪表盘，而是“当前状态 + 当前推进”的轻聚焦面。

### 模块结构

推荐拆成左右两个主卡：

1. `当前状态卡`
2. `下一步行动卡`

### 4.4.1 当前状态卡

#### 功能目标

让用户一眼知道自己处于什么状态，并且感觉被理解。

#### 必备内容

- 模块标签：`当前状态`
- 情绪或状态主词
- 一句状态说明
- 当前聚焦主题
- 学习进度概览

#### 文案原则

状态词应：

- 简短
- 温和
- 可读

例如：

- 平静
- 有一点乱
- 专注
- 有些焦虑

状态说明必须是陪伴型，不是诊断型。  
不要写成：

- 你的效率偏低
- 你当前状态较差

应写成：

- 你现在不需要一下子想清全部
- 这已经比刚才更聚焦了

#### 视觉原则

- 状态词可以比正文更大
- 蓝色只轻量参与，不应变成整块纯蓝卡
- 允许加入轻量圆环、节律线或弱图形

### 4.4.2 下一步行动卡

#### 功能目标

把当前主线压缩成一个明确、可执行、不压迫的动作。

#### 必备内容

- 模块标签：`下一步行动`
- 行动标题
- 一句补充说明
- 进度条或轻进展提示
- `带回对话` 按钮

#### 行动标题规则

行动标题必须：

- 可执行
- 粒度小
- 不模糊

好例子：

- 先写第 1 句
- 开口复述 3 句例句
- 决定今天几点开始

坏例子：

- 继续努力
- 推进主任务
- 做学习计划

#### 按钮逻辑

`带回对话` 的本质是：

- 把结构化行动转为自然语言
- 填入输入框
- 不自动发送

#### 视觉原则

- 这是 Hero 中推进感最强的模块
- 但不能用过强颜色造成压迫
- 可用蓝色进度作为唯一强强调

---

## 4.5 Conversation Stream

### 模块目标

这是此刻页最核心的模块。  
用户与 Echo 的关系建立、澄清、安抚、推进，全部发生在这里。

### 模块组成

- 区域标题
- 快捷提示条
- 对话时间线

### 对话结构

每条消息包含：

- 角色
- 内容
- 时间

角色分为：

- 用户
- Echo
- 特殊情况下的 System

### 设计原则

- 一眼看出谁在说话
- Echo 的气泡更柔和
- 用户的气泡更简洁
- 时间弱化
- 行间距舒适

### 行为原则

- 默认只展示最近一段
- 新消息进入时自动滚动到底部
- 发送中允许出现 typing 状态

### 快捷提示条

#### 功能目标

减少输入起步成本，让用户更容易继续对话。

#### 推荐文案

- 继续下一步
- 我有点焦虑
- 继续当前线索
- 先帮我缩小一点

#### 限制

- 只承接当前主线
- 不做复杂推荐
- 数量建议 `2 - 4`

### 空状态

当没有历史对话时，应显示轻量空状态：

- 只用一句话引导
- 不要做复杂插画
- 不要营造“空系统”的冷感

推荐：

- 从你现在最想说的一件事开始

---

## 4.6 Composer 输入区

### 模块目标

让用户能低阻力地继续表达。

### 模块组成

- 输入框
- 发送按钮
- 可选朗读按钮
- 可选语音入口

### 输入框要求

- 优先单行
- placeholder 温和
- 不写成命令式

推荐 placeholder：

- 继续输入你现在的状态
- 说说你现在最想先理清的一件事

### 主按钮要求

- 唯一主按钮是 `发送`
- 清晰、稳定、可预期

### 辅助按钮要求

- `朗读最新回复`
- `语音`

辅助按钮必须弱于发送按钮。

### 状态要求

需要支持：

- 默认
- 聚焦
- 发送中
- 禁用
- 错误恢复

---

## 4.7 右侧 Status Workbench

### 模块目标

把后端状态聚合成用户能扫读的轻工作台。  
它的职责是辅助推进，不是替代对话。

### 模块数量控制

建议维持 `4 - 5` 张卡。  
禁止无限扩张。

推荐顺序：

1. 当前状态
2. 专注
3. 下一步行动
4. 学习线
5. 反思趋势 或 记忆概览

---

## 4.7.1 当前状态卡

### 目标

在右栏复述当前状态，但更偏摘要，不和 Hero 重复竞争。

### 内容

- 状态名称
- 状态说明
- 节律线 / 波形 / 轻图形

### 原则

- 图形轻
- 不是医疗监测感
- 不是强科技感

---

## 4.7.2 专注卡

### 目标

展示当前主线聚焦程度。

### 内容

- 当前聚焦主题
- 百分比或进展值
- 进度条

### 原则

- 进度条是核心
- 文案简短
- 不做复杂说明

---

## 4.7.3 下一步行动卡

### 目标

在右栏再次给出当前行动锚点，方便随时带回对话。

### 内容

- 行动标题
- 行动说明
- `带回对话` 按钮

### 原则

- 全页只能有一个主行动锚
- 这里不能同时出现多个建议动作

---

## 4.7.4 学习线卡

### 目标

展示当前学习主题和所在步骤。

### 内容

- 学习主题
- 当前步骤一句话
- 分段进度

### 原则

- 优先图形化
- 不写长段解释
- 只显示当前主线

---

## 4.7.5 反思趋势 / 记忆概览卡

### 目标

提供长期连续性的轻提醒。

### 二选一原则

默认只放一个，不同时放两个重模块。

可选：

- 反思趋势
- 记忆概览

### 反思趋势卡内容

- 近期趋势词
- 一句摘要

### 记忆概览卡内容

- 当前被记住的主题
- 标签 / 片段数量

### 原则

- 只做轻提示
- 不做专题页替代品

---

## 5. 状态映射规范

## 5.1 情绪状态映射

情绪状态至少包含以下层：

- `neutral`
- `focused`
- `motivated`
- `anxious`
- `distracted`

映射维度：

- 主词
- 辅助说明
- 图形节奏
- 蓝色强度

示例：

### neutral

- 主词：平静
- 图形：低波动
- 强调：低

### focused

- 主词：专注
- 图形：节奏稳定
- 强调：中

### anxious

- 主词：有些焦虑
- 图形：频率略高
- 强调：中高，但不刺眼

---

## 5.2 任务状态映射

此刻页仅消费动作状态，不负责复杂管理。

需要映射：

- `pending`
- `active`
- `done`
- `dismissed`

页面表现原则：

- 当前行动只读为主
- 不在此刻页直接做复杂编辑
- done 和 dismissed 不在主视图占主位

---

## 5.3 学习状态映射

学习状态以当前主线为唯一来源。

页面只关心：

- 当前主题
- 当前步骤
- 总步数
- 进行中位置

页面不显示：

- 多主题并发
- 大量历史步骤

---

## 6. 交互流规范

## 6.1 发送消息

流程：

1. 用户输入内容
2. 点击发送或回车
3. 用户消息上屏
4. Echo typing
5. Echo 回复上屏
6. 页面状态刷新
7. Hero 和右栏同步更新

### 要求

- typing 状态必须轻
- 不让用户怀疑系统是否接收成功

---

## 6.2 点击快捷提示

流程：

1. 用户点击快捷提示
2. 文案填入输入框或直接进入对话意图
3. 用户继续发送

### 要求

- 默认推荐填入而不是强制自动发送
- 保留用户控制感

---

## 6.3 点击带回对话

流程：

1. 从结构化数据生成自然语言
2. 填入输入框
3. 光标聚焦
4. 等待用户发送

### 要求

- 不自动发送
- 文本必须自然，不像系统码

---

## 6.4 页面刷新

当此刻页相关数据变化后：

- Hero 刷新
- 对话区域刷新
- 右栏刷新

刷新原则：

- 无感
- 不闪屏
- 不打断输入

---

## 7. 文案规范

### 7.1 文案层级

页面文案分为：

- 页面级文案
- 模块标题
- 行动文案
- 陪伴型说明文案
- 系统反馈文案

### 7.2 语气要求

所有文案必须满足：

- 温和
- 简洁
- 具体
- 不评判
- 不命令

### 7.3 禁止文案

- 立即执行
- 你失败了
- 效率不足
- 请严格完成

---

## 8. 空状态、加载态、错误态

## 8.1 空状态

页面空状态只做轻引导。

不做：

- 大插画
- 强营销提示
- 长段说明

## 8.2 加载态

优先使用骨架屏。

骨架屏位置：

- Hero 卡
- 对话气泡
- 右栏卡片

## 8.3 错误态

错误反馈必须克制、可恢复。

推荐：

- 当前同步暂时不可用
- 刚才这条消息还没有成功发出

避免：

- 系统异常
- 未知错误

## 9. 前端组件建议

推荐组件拆分：

```text
NowPage
├─ NowPageHeader
├─ NowHeroSection
│  ├─ CurrentStateHeroCard
│  └─ NextActionHeroCard
├─ ConversationBoard
│  ├─ QuickPromptBar
│  ├─ ConversationTimeline
│  └─ Composer
└─ StatusWorkbench
   ├─ CurrentStateCard
   ├─ FocusCard
   ├─ NextActionCard
   ├─ LearningCard
   └─ ReflectionOrMemoryCard
```

组件原则：

- UI 与数据逻辑尽量分离
- 每张卡片单一职责
- 带回对话逻辑统一复用

---

## 10. 验收标准

此刻页设计通过的最低标准：

1. 用户第一眼能看懂当前页在做什么
2. 对话流明显是核心，不是装饰
3. 下一步行动明确，但不压迫
4. 右栏信息有用且克制
5. 页面整体有呼吸感，不拥挤
6. 文案语气温和，不像任务管理器
7. 数据刷新后界面变化自然
8. 打开页面时能感到“我可以继续”，而不是“我又要被安排”

---

## 11. 最终一句话定义

`此刻页` 不是一个聊天页，也不是一个任务页。  
它是 Echo 用来接住用户当下、维持连续关系、并把下一步轻轻推出来的主页面。
